# 支持指南

## 获取帮助

请先阅读 [README](README.md)、现有 [GitHub Issues](https://github.com/KasumiKitsune/PhonTracer/issues) 和相关工程文档。确认问题未被记录后，可选择“错误报告”或“功能建议”模板新建 Issue。

一个可复现的支持请求应包含：

- PhonTracer/PhonRec 版本与下载的资产文件名；
- 操作系统版本与 CPU 架构（Windows x64、Windows ARM64 或 macOS Apple Silicon）；
- 使用的入口（主程序、Toolkit、CLI、PhonRec 完整模式或独立模式）；
- 最短复现步骤、预期结果与实际结果；
- 完整错误文字、退出码和必要的脱敏截图；
- 问题是否能在新建空白工程中复现。

## 数据安全

GitHub Issue 默认公开。请勿上传未获授权的录音、真实发音人信息、完整 `.teproj`、访问令牌、私有路径或机构敏感数据。若必须提供样本，请创建最小化、合成或充分脱敏的复现材料。

疑似漏洞、路径越界、令牌泄露或恶意工程问题不要作为普通 Issue 公开；请按 [SECURITY.md](SECURITY.md) 私下报告。

## 支持边界

- 优先支持仓库最新发布版本和当前 `main`；
- 第三方 Python 环境、改装打包、非官方插件和用户自定义脚本需先在干净环境中复现；
- 声学分析结果需要研究者人工复核，项目不提供医学、司法或临床结论；
- 未在发布矩阵中列出的系统与架构属于尽力支持范围。

维护者会按可复现性、影响范围和可用时间处理请求，不承诺固定响应时限。
