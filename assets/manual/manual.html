01 / 核心概念

1. 核心概念与术语速查

基频 (Fundamental Frequency, F0)

物理声学中反映声音高低的指标，单位是赫兹 (Hz)。语音学中常说的“声调起伏”，在声学层面主要就是通过提取 F0 曲线来体现的。

VAD (语音活动检测)

计算机算法，自动判断录音中哪段有人说话，哪段是环境底噪。PhonTracer 处理长录音的第一步，就是靠它将大段录音“切萝卜”般分成有声片段。

发音人 (Speaker)

实验中录音的具体对象。软件支持同时管理多个发音人，各自的音频、修改进度和参数彼此隔离，互不干扰。

词项 (Item) 与音节 (Syllable)

数据处理的基本层级。“词项”对应字表中的一行（一个词、短语或句子），软件会根据规则将其拆分成“音节”，并在每个音节内部均匀采集基频数据（默认 11 点）。

TextGrid 格式

语音学界常用分析软件 Praat 的标准标注格式，记录录音的时间段与字词对应关系。PhonTracer 支持无缝导入和导出。

T 值 (T-value)

消除发音人个体差异的数据归一化方法。通过对数转换，将不同声音高低范围映射到 0 到 5 的区间，使不同人声调走势可以直接对比。

03 / 界面漫游

3. 软件主界面漫游指南

主界面布局划分为三个明确的区域，分别在宏观管理、精细微调和成果存储方面发挥作用：

左侧面板：全局指挥中心

管理项目的运行状态、算法引擎参数与核心分析模式。

  • 发音人管理： 支持添加、切换与隔离多发音人工作区。开启“统一字表”后一次导入全局应用，避免多受试者实验的重复劳动。
  • 分析模式切换 (核心功能)： 新增“声调/F0”与“共振峰/F1-F2”一键分析模式控件。切换模式时，系统会自动在后台独立缓存对应模式下的所有边界状态，确保已修剪的数据绝对不会丢失。
  • 全局参数折叠面板： 涵盖基频最低/最高限值、清浊阈值、共振峰分析上限及采样策略等算法控制项，一处修改，全局生效。

中间面板：可视化精修车间

显示被选中词项的声谱图、波形图及实时计算线条，提供超高精度的声学修正能力。

  • 声谱叠加线： F0 模式下，声谱图叠加蓝色 F0 曲线；切换为共振峰模式后，纵轴频率自适应共振峰分析上限，并在图上高亮强调橘色 (F1) 和绿色 (F2) 的共振峰轨迹散点，自动隐藏基频纵轴。
  • 起止红线与分节蓝线： 红色线条表示词项边界，蓝色虚线表示内部音节交界。可通过鼠标左右无障碍拖拽进行物理精细调整。
  • 橡皮擦与高精度剔除点 (滚轮调节)：
    • F0 模式： “橡皮擦”工具按左键拖动批量刷除局部杂点，抹平倍频跳变。
    • 共振峰模式： “剔除点”工具采用点定位算法，每次仅擦除离鼠标最近的单个共振峰点，防止时间帧序列大面积断档。
    • 智能交互： 在任意模式下，均可通过鼠标滚轮实时无缝缩放十字准星的感应半径。
  • 多维播放控制与淡黄色播放选区： 音频控制条集成了播放、停止、循环、当前字等工作模式。在声谱图空白区按住左键并拖曳，可拉出淡黄色的播放选区，配合循环模式可在微观分割处反复重播，高效听辨元音过渡。

右侧面板：项目树与成果仓库

用于数据树状归档、声学异常警示以及自定义规则导出。

  • 项目树状态短标签： 项目树右侧的列表会显示简短的状态标签，例如将冗长的异常警告或忽略原因压缩为更短的文本标签（如 +N 等），避免过长的中文文案挤压您的界面空间，帮您快速扫视状态。
  • 智能筛查与“需检查”视图：
    • F0 模式下： 自动收集有声帧比率偏低、异常高低八度跳频的词项。在声谱图上，被判定为疑似倍频/半频/噪声的异常频点也会以亮红色高亮警示。
    • 共振峰模式下： 收集 F1/F2 错位 (F2 ≤ F1) 比例偏高、有效帧数过少的词项。
  • 右键属性窗口（底层核对）： 怀疑页面显示和真实保存的不一致？您可以右键任何词项或分组选择“属性”。它不仅会显示词项的高级元数据，还会列出该词项实际绑定的独立参数值，明确告诉您它用的是“条目值”还是“发音人默认值”，是排错的最佳帮手。
  • 右键忽略单项警告： 在项目树上右键点击警告条目，选择“忽略警告”可绕过该词项的异常视图筛查。适用于发音人清音化或特定方言音系造成的缺失现象。
  • 条目级参数隔离机制： 当你在左侧仅仅调整当前词项的独立参数（比如调某个词的基频上限）时，这套“局部参数”不会污染同一个发音人的其他组或词项。
  • 一键导出与 AnomalyWarningDialog 异常拦截： 导出数据时，若项目中仍存留未处理的异常，系统会弹出滚动拦截弹窗，直观罗列警告详情，双击可快速跳转微调。用户亦可一键“忽略警告并强行导出”。
04 / 核心工作流

4. 两大核心工作流与发音人管理

理论保障 字表格式（普通字表与高级字表）与音节拆分规则

PhonTracer 支持导入普通文本字表（.txt）、带结构的 CSV，以及其核心的 高级字表 (.ptwl) 格式。

普通字表与高级字表 (.ptwl) 的区别

  • 普通字表 (.txt)： 只有简单的组名与词项。你用方括号包住的词组只能当成视觉上的名字，不会自动成为数据的科研结构元数据。它适合于只想快速对齐音频而不打算按词性、调类做复杂组合分析的场景。
  • 高级字表 (.ptwl)： 这是一种带有强大学术结构化数据的 JSON 载体。它不仅保存了你的字词，还能为每组、每个词附加：组备注、组标签、词项备注、词项标签、别名、自定义字段（比如“词频等级”、“变调条件”、“结构类型”），以及人工复核状态。
  • 元数据穿透机制： 一旦你在主程序导入了 .ptwl 高级字表，这套元数据就会如影随形地“穿透”并永远附加在每个被切分出来的声音条目上！后续不管是你画图、导出 Excel 还是写脚本，你都能随意按这些自定义标签对声音进行分类对比，这也是普通文本字表永远做不到的。
导入 .ptwl 时的重点注意： 虽然导入 .ptwl 时左侧窗口仍然会转换成大家熟悉的带组名文本让您预览，但内部其实绑定着完整的元数据！只要您别用键盘瞎乱删改这个预览文本框里的内容，导入生效后所有高级元数据就会稳稳注入您的音频工程中。

字表格式范例

【普通话】
北京
上海

【英语】
bro/ther
sys/tem

说明

可使用中文括号 【分组名】、英文括号 [分组名] 或 Markdown 样式的 # 分组名 自定义分组。同一行写多个词时可采用空格、逗号或顿号。

警告：字表中只推荐保留最纯净的文本，不推荐包含 001_北京 类似标记，以免在斜杠拆分时干扰规则判定。

2. 中文汉字自动拆分 北京 北、京

流程 A 长音频工作流：大批量有序采集

适用于发音人连续不间断朗读长字表的录音场景。程序通过自动 VAD 将大文件“一刀切切分”，并自动将词项按照顺序扣在有声片段上。

可视化切分器的操控与手动调整

VAD 有时可能出现断句错误、噪音多检或吞字。调出“可视化切分器”，即可通过图形界面修正不平整的切片：

1. 删除误检 对准误检出的噪音区域段点击鼠标右键，直接将其移除。
2. 手动框选 在空白没有被自动标上的波形区域右键，可选择直接手动新增片段。
3. 边界微调 直接用鼠标左右拖拽有色边界，对准首尾起止点重新对齐。
TextGrid 无缝对接： 如果您已经有经由 Praat 人工细筛过的标注 TextGrid 文件，可以直接导入，程序会优先调用其层级（支持 words 词边界层、chars 音节层以及 groups 分组层），自动跳过不确定的 VAD 自动切分阶段。

流程 B 批量音频工作流：零碎小文件整合

适用于已被分切成若干独立小文件的录音材料。直接导入文件夹即可全量处理。为了精确标注，推荐导入字表对音频和词项进行“联姻”。

模式 1. 模糊匹配

当文件名本身杂乱，包含序号或临时后缀（例如：001-北京(重录).wav）时，系统将智能剔除无关字符，主动寻找最接近字表的拼音或汉字。

001-北京(重录).wav → 北京
模式 2. 顺序匹配

如果文件名完全不包含词语信息，但在导入排序时已和字表顺序严格对应。系统将放弃文本识别，按照队列顺序强行生套匹配。

1.wav, 2.wav → 对应字表 1、2 行

发音人多维管理与“统一字表”

研究中往往涉及多位受试发音人。左侧的发音人面板就是工作隔离区。每一位发音人都相当于一份独立且安全的沙盒进度。 如果所有受试发音人朗读的是同一套材料，请务必在面板顶部启用 “统一字表”。开启该功能后，仅需在第一个发音人中导入一次字表，切换或添加其他发音人导入音频时即可直接继承字表以及相关的分音节规则，大幅提升大样本处理效率。

05 / 算法参数

5. 算法原理与关键参数调优

为了使导出的声调和元音轨迹具有学术说服力，必须知晓每一项参数底层的运作机理，以便在各种噪杂环境或极端发音特征下进行弹性干预。

声学边界定位的两级切分路径

宏观定位（选地盘阶段） 长音频 VAD 阶段。算法通过分析录音的声音强度基底，计算动态噪声门槛，并标记出存在浊音振动的区域，界定“这个时间段对应字表第几行”。
微观定位（找核心阶段） 由于阻碍音、爆破音没有基频表现，真正的声调仅依附于元音与响辅音上。微观阶段即综合声强峰值、声能衰减落差去搜寻元音的分布边界，作为最终数据采样的提取区。

5.2 智能估测：告别盲猜 F0 范围

系统内置了高度精准的基频统计估计模块，点击“估计”按钮可瞬间对已加载的音频样本进行整体扫描，生成发音人的真实物理声带特征库。

① 有声区间粗筛与剪枝 以 50Hz 到 700Hz 极宽频段扫描浊音帧，并自动剪除时长短于 30 毫秒的非稳态气流帧，彻底隔离声门呼吸和辅音残留的干扰。
② 倍频抑制与跳变平滑 算法实时计算相邻时序帧的 F0 变动率。一旦发生大于 20% 的急剧跳变（生理声带无法实现），系统会进行片段权重切断，抑制倍频/半频造成的统计偏差。
③ 分位数区间多档推荐 基于 F0 浊音纯净频段点集，计算 P5、P10、P50（中位）、P90、P95 等百分位数，生成保守范围 (Conservative P5-P95)、推荐范围 (Recommended P10-P90) 和精细范围 (Fine P15-P85) 三档建议。
音节边界保护机制： 应用这些推荐的基频区间范围时，软件会自动采用“局部音高重算更新”策略，仅刷新声学采样轨迹，最大程度强行保护用户已经手动精细调校完毕的红蓝起止边界线不受重置影响。

5.3 声调 (F0) 核心参数详解

pts (采样点数) 默认：11

在每个音节内部均匀采集 F0 频点的个数。通常声调起伏研究设置为 11 点已经能精准描摹轮廓；若需做更微观的时序过渡变化研究，可调大至 20 之间。

db (能量下降值) 默认：60

微观划分元音核心区域的尺子。数值越大对能量表征越宽容，越容易把一些弱声尾音或送气摩擦声划入范围；数值越小则框线收缩得越紧，边界较窄。

skip_front (排除声母) 默认：0 秒

并不简单粗暴割裂音频，而是给“元音起点搜寻算法”强行划出时间盲区。对于爆破送气音重的辅音字母开头，设为 0.05s - 0.1s 可以规避声响干扰。

Pitch Floor/Ceiling 默认：75 - 600 Hz

算法测高下限。嗓音浑厚的男低音可调至 50/60；高亢女高音或儿童发音，Ceiling 可提至高值，Floor 设得过低容易错误识别成环境的低频嗡嗡声。

voicing_threshold 默认：0.25

判定发声是否有声带振动的阈值。数值设低较易识别一些低沉、气声或嘎吱音 (creaky voice)；数值越高越保守，不满足阈值的倾向于判零值。

trim_silence (边缘切除) 默认开启

在词的边缘把低于 -50 dB 的极度微弱静音区域进行精细过滤。

F0 基频提取算法说明

Praat 自相关算法 使用最经典的自相关 (Autocorrelation) 算法，是目前学术界进行语音声学分析最认可且具有高解释度的基石算法。
对于嘎裂声、严重不规则振动或弱周期信号，无法可靠测量时会保留为缺失值。请结合声谱图人工复核，不要为了得到连续曲线而强行补齐。

5.4 共振峰 (F1/F2) 分析模式与声道物理声学大解密

进入“共振峰/F1-F2”模式，PhonTracer 将开启基于声道形状调谐的元音检测通路。声道声学理论指出，母音特征由前两个共振峰 (Formants) 空间映射表征：

第一共振峰 (F1) —— 与舌位高低 (口腔开闭) 呈负相关 舌位越高（如闭元音 /i/, /u/），声道前段越狭窄，F1 频率值越低；舌位越低（如大开元音 /a/），开度最大，F1 频率显著抬高。
第二共振峰 (F2) —— 与舌位前后 (前元音/后元音) 呈正相关 舌位越靠前（如前元音 /i/），前腔共振容积压缩，F2 频率越高；舌位越靠后（如后元音 /u/, /o/），舌根隆起咽喉共鸣，F2 显著降低。

PhonTracer 采用经典的 Praat Burg 法 (praat_burg 引擎) 进行共振峰时序散点定位。精细分析需配置以下四个底层核心控制参数：

  • 追踪共振峰数量与分析上限 (formant_max_hz)： 默认追踪 5 个。该值是 Praat Burg 的搜索上限，不是 F2 数值上限。成年男性通常可从 5000 Hz 附近起步；成年女性通常可从 5500 Hz 附近起步；儿童或较短声道发音人可能需要 8000 Hz 左右。设错会改变 LPC 极点分配，使 F1/F2 换轨或误跟到高阶共振峰。
  • 分析窗长 (formant_window_length)： 默认 0.025s (25毫秒)，在时间与频率分辨率间取得折中。
  • 预加重起始频率 (formant_pre_emphasis)： 默认 50 Hz。对随频程高阻衰减 (每倍频程-6dB) 的高阶共振峰信号进行能量补偿，提升寻峰精度。
  • 共振峰采样策略 (formant_sample_strategy)：
    • 整段11点： 均匀取样 11 点，适合双元音滑动过渡或观察辅音协同发音 (coarticulation) 转瞬变异。
    • 中段均值： 仅计算音节中间 1/3 长度段均值，极大程度过滤首尾辅音过渡段的声学非稳态区，为绘制高稳定度的单发音人元音舌位图而设计。
抗 F1/F2 错位 (Swap Error) 与自动推荐机制： 紧元音（如 /i/）因舌位高且前，声学能量易堆积混淆，致使 F2 错误下跌跌入 F1。PhonTracer 估计工具提供多档参数方案：
  • 系统推荐档： 均衡平顺，通用性高。
  • 低上限复核档： 采用较长的分析窗长与偏低的分析上限，用于复核高上限是否漏掉低位 F1/F2。它不是对所有错位问题都更优，仍需结合声谱图判断。
  • 高分辨率档： 使用极短的采样窗长，原汁原味还原母音内部的频率瞬态起伏。

7.2 声调 (F0) 专属科学图表家族

声调轮廓图 (Tone Contour) 即经典的声学格局图。横轴支持等距采样点 (normalized) 或物理时长 (duration)；曲线可展示为仅组均值、均值+浅色个体细线 (方便看组内发音变异)、或均值+标准差置信半透明阴影。
声调分布图 (Tone Distribution) 定量表征发音的稳定性与离散程度。支持选择科学箱线图或小提琴图，展现每一物理点位跨样本的概率分布、起-中-终经典三点分布对比、以及各组发音一致性的变异系数 (CV) 柱状图。
时序密度图 (Pitch Time Density) 基于二维高斯核密度估计 (2D KDE) 拟合生成时序密度热力图。支持调整核带宽 (bandwidth)。由于核估计对突变噪点极度敏感，提供 5%-95% 分位数截断以过滤低频噪声，集中展示发音音域团。
数据质量检查图 (QC Check) 论文导出前的“大体检排雷器”。模块细分为：
① 逐项基频叠加图，正常画为蓝细线，有大段 NaN 的异常曲线用红色虚线醒目标出，方便精准回溯。
② 检出率箱线图，设 60% 检出率红虚警戒线。
③ 调域误差棒图，直观锁定发生半频/倍频错误的离群发音人。
声调组别概览图 (Group Overview Heatmap) 对大样本声调语料库或方言调查大批量字表的快速概览。生成全局采样点上的均值矩阵热图 (Hz 下为 viridis，T值下为红蓝 RdYlBu_r 渐变) 或标准差矩阵热图 (Reds 渐变，红色越深变异度越大)，一目了然定位不稳定声学区段。

7.3 共振峰 (F1/F2) 专属科学图表家族

若在左侧开启“共振峰/F1-F2”分析模式，科学图表编辑器会自动切换装载专属绘图模块，支持以下三把高段位元音分析利器：

① 元音共振峰空间图 (Vowel Space Chart) —— 学术论文最核心图表！ 图表遵循语音物理声学坐标轴反向惯例 (Academic Inversion)：
  • 横轴 F2 反向： 坐标数值自右向左递增（越靠右值越小，代表后舌位；越靠左值越大，代表前舌位）。
  • 纵轴 F1 反向： 坐标数值自上而下递增（越靠上值越小，代表高舌位/口腔闭；越靠下值越大，代表低舌位/口腔开）。

该经典的镜像反转轴设计，将纯声学赫兹数据还原为与人类发音时口腔物理空间完全重合的“母音舌位图”（左上 /i/、右上 /u/、下方 /a/）。支持一键绘制 50% 或 95% 置信度数据椭圆 (Confidence Ellipse)，清晰刻画群体母音范围；亦可选择叠加强调时间演变渐变线，展现音节内部元音滑移走势。

② 共振峰时序轨迹图 (Trajectory) 在等距时间轴上，同时分轨追踪 F1、F2 和 F3 的频率演变。在解析协同发声、双元音滑移轨迹中起到核心判定作用。
③ 共振峰密度分布图 (Density KDE Map) & 概览热图 大样本量下散点会重叠发黑，KDE 密度图利用二维高斯密度模型，勾勒出深邃美丽的“热力星系云团”，直观指明元音空间物理分布的最核心聚拢重心区。概览热图则可一站式快速概览海量音节在稳态 F1/F2 上的数值及变异差表现。

8. 进阶：Toolkit 音频前处理工具箱

当面对混乱无序的实验音频时（如时长一个多小时且语速多变的整轨、或是数百个命名杂乱的碎片文件），建议在载入核心软件前，使用 Toolkit 进行批量化的整理。

8.1 合并音频（碎片缝合）

如果你想用长音频模式的连贯工作流，但手头全是一堆碎音频，用它！

  1. 把多个 .wav 或 .mp3 拖进去。
  2. 在面板内用键盘 Delete 键删除错误项，拖动调整队列先后顺序。
  3. 设定拼接时的“静音间隔”（默认 0.5 秒，这能帮助 VAD 算法极好地认出边界）。
  4. 点击合并。软件不仅会把它们无缝拼接，还会自动统一重采样到标准的 44100 Hz，输出一个纯净无暇的长 WAV 文件。
文件名排序优化： “导入字表自动排序”功能，只要录音文件名中包含字表里的词，程序就会将小文件按字表重排，无法对应的音频扔到尾端，极大加速后续处理。

8.2 拆分长音频（一键切块）

假设你在做语料库整理，要把一长段录音切成几百个带标号的小文件。

  1. 载入长录音文件。
  2. 粘贴或导入字表（一行一个词）。文件解析内置了自动剥离文件开头 UTF-8 BOM 字符的逻辑，防止因可见/不可见 BOM 字符导致首行匹配失效。
  3. 点击“一键匹配”，软件会自动进行宏观的语音活动检测（VAD）。如果发现切得不对，可以进入段落编辑器，利用鼠标滚轮进行波形图横向滚动，或使用 Ctrl + 滚轮对波形进行水平缩放，精细调整边界、点击播放听辨、删除误检段或添加手动段落。
  4. 设定边缘缓冲时间（比如默认留 0.1 秒的白头白尾，听起来更自然）。
  5. 点击拆分，你会瞬间得到一个目录，里面的文件被完美命名为 001_北京.wav、002_上海.wav... 文件名里如果存在系统不允许的非法字符，软件会自动替换为合法下划线，绝不报错卡死。

8.3 高级字表 (.ptwl) 可视化编辑器

Toolkit 不再只是一个切音频的工具，现在它也是你的实验材料中央厨房。在“字表”标签页中，您可以直接构建带丰富科研元数据的 .ptwl 文件。

  • 丰富元数据录入： 左侧管理组名与组备注/标签，右侧可细致填写每个词的各种标签、别名，以及最关键的自定义科研字段。
  • 智能导入导出互转： 你可以随时将传统的 .txt 文本字表或 .csv 文件导入成高级字表；如果是外部人员填好的带有无数列数据的 Excel（需另存为CSV），工具箱会把未知列自动转换为 item_meta（自定义字段）！反之也可以导出为 CSV。
  • 自动对接： 编辑完成后保存，或点击“同步到音频拆分”，您可以马上使用这个高级字表去跑切分。

8.4 自定义脚本沙箱：探索性科研与受控 AI 协作

如果你想画的图 PhonTracer 主程序里没有怎么办？在 Toolkit 的“自定义脚本”页，我们为您准备了一个强大的研究沙箱！它不是一个允许随意执行任意 Python 代码的危险控制台，而是一个受控、隔离、可归档的协作分析层。

  1. 在主程序完成所有的红蓝线纠错工作，并导出 .teproj 完整工程。
  2. 打开 Toolkit，载入这个工程文件，并切到“自定义脚本”页。
  3. 点击底部的“复制提示词”。软件会瞬间把您工程的数据格式（有哪些标签、多长、几个发音人）整理成给 AI 的极度清晰的提示词！
  4. 把这串提示词发给您的 AI（比如 ChatGPT、Claude），告诉它你想研究什么。AI 会在安全沙箱的允许范围内（默认开放 numpy、scipy、matplotlib，禁止系统访问和未允许的重型库如 seaborn/pandas/gaussian_kde）为您编写一段 def run(ctx): 函数代码。
  5. 将代码粘贴回脚本编辑器中，点击“运行”。在后台安全跑完后，您能立刻看到生成的图表或 CSV 数据表。
  6. 科研归档机制： 如果运行成功，您可以选择将此次执行记录写入到该 .teproj 内部。之后当您生成报告时，这段脚本的代码指纹、运行时间及结果会一并附在你的论文归档中。

8.5 工程包结构预览与报告导出

Toolkit 中内置了高效的工程操作模块，专为无需载入主程序的大型工程分析而设计：

  • 结构预览： 载入 .teproj 工程包后，工具箱会以极轻量级的方式解析其内部结构，快速呈现工程版本、发音人名册、各发音人工作模式、音频数量和已分析项目总量。这为数据交付与多人合作工程审查提供了极大便利。
  • 研究报告智能生成： 您可以在此直接导出带有 SHA-256 校验和以及高级脚本记录的科研方法报告，现在导出界面还配备了实时的阶段进度提示。
  • 工程包转换另存： 支持将打包好的 .teproj 另存为标准的 .zip 压缩包，以便于研究人员直接解包并读取底层的 project.json 结构或提取缓存在里面的音频副本及声学缓存。

8.6 界面导入与拖拽稳定性说明

在 Windows + Tkinter 的特定多线程运行环境下，直接使用窗口级底层鼠标拖放（如 windnd 库）极易与 Python 的 GIL（全局解释器锁）发生冲突，甚至在导入大规模无序文件时导致程序崩溃。为了保证实验软件的高度稳定性，在新版 Toolkit 中：

  • 稳定路径导入： 关闭了窗口级的外部文件直接拖入入口。推荐用户使用界面中的“添加音频”、“选择长音频”或“选择工程文件”按钮，通过标准文件浏览器对话框进行文件导入。
  • 列表拖动排序： 导入列表后，其内部的拖拽排序、Delete 键删除选中项等交互机制依然被完美保留且稳定运行。
安全锁提示： 外部直接拖入可能会造成多线程死锁。请统一使用界面的文件对话框安全载入材料。
09 / 自动化与命令行

9. 极客批处理与 AI 助手：CLI 命令行指南

为了满足极客用户、Python 程序联动以及 AI Agent 自动化（如 Cursor, Trae, Codex）的需求，PhonTracer 内置了功能完备的纯文本交互 CLI 引擎。所有的命令执行状态及错误分析均标准化为结构化 JSON 格式输出，完美支持管道调用。

CLI 高频运行与 Agent 稳定性设计：
  • 单指令单次直接执行： 支持 python cli.py status 或 PhonTracerCLI.exe --help 等无窗口非交互直接调用。若参数为已知内置命令，系统会在执行完毕后将结果以 JSON 回显输出在标准输出上，然后安全退出。
  • Agent 启动容错与多进程隔离： Windows 打包版底层集成了 multiprocessing.freeze_support()，并在参数层过滤忽略以 --multiprocessing-* 开头的系统级内部启动参数。这彻底规避了 Windows 下进行多进程基频重算时，子进程被误读为用户交互命令输入的循环死锁崩溃。

常用 CLI 指令词典

全局设置与声学配置
  • lang zh / lang en : 设定中英文回显环境。
  • speakers / add_speaker : 查看或新增发音人隔离工作区。
  • set_params pts=11 db=60 pitch_floor=75 : 设定声调提取参数。
  • set_params analysis_mode=formant formant_max_hz=5500 : 设定共振峰模式参数。
  • autosave on / off / now : 查看、启闭或强制原子备份写入。
  • agent_guide : 返回包含 AI 交互守则指南 of 结构化 JSON。
数据处理与转换
  • project_import / project_export : 导入或导出标准 .teproj 工程包（核心交付命令）。
  • load_long / load_batch : 导入长录音或小片段集。
  • apply_wordlist "表.txt" fuzzy/order : 模糊或顺序对齐字表。
  • detect_f0 recommended/conservative/fine : 估测基频，一键应用并采用局部重算保护手工边界。
  • import_batch_and_export <dir> <out.teproj> : 批量扫描 WAV + TextGrid 对，自动对齐声学分析并打包。
  • list_items warnings : 列出所有有声帧比率偏低或报错的词。
  • modify_bounds 3 1.250 1.820 : 强行调整指定词起止时间。
脚本管理与受控执行
  • list_scripts : 列出当前本地保存的所有自定义脚本。
  • script_info "脚本名" : 查看指定脚本的源码与使用说明。
  • script_save / script_delete : 脚本的本地生命周期管理命令。
  • script_import / script_export : 导入与导出脚本文件。
  • script_prompt / generate_prompt : 在命令行直接生成 AI 画图提示词。
  • run_script "脚本ID" timeout=30 : 在受控沙箱内静默执行目标科研脚本，返回日志并在工程中归档。

10 / 疑难解答

10. 常见故障排雷与应急指南

症状： 软件提示某些词“缺失 (Missing)”，或者后面的词段全部产生了位移偏差。
急救： 造成该问题通常是因为发音人出现了吞音（两个词连续朗读且无边界停顿）或者录音中途掺杂了咳嗽、翻书等严重白噪音被误判为发音片段。此时双击打开界面中间的“可视化切分器”，在误检的有声块上右键选择直接删除，或在缺失片段的空白波形区域右键选择添加。确认排序并重新应用对齐，即可复原。
症状： 表格和可视化绘图中曲线多处空缺，均值受到极大扭曲。
急救： 首先确认该片段是否是真正的浊音（声带产生明显振动），清辅音由于物理原理本就不含基频。如果是正常浊音但算法不予捕捉，可以谨慎降低 Pitch Floor (最低基频) 和 Voicing Threshold (清浊判定阈值)，再结合声谱图人工复核。如果片段存在嘎裂声、严重不规则振动或弱周期信号，应保留缺失值，不要为了得到连续曲线而强行补齐。
症状： 数据出现物理学上的倍频或半频判定错误。
急救： 男低音易产生高飞杂点，女高音易产生掉落半频点。如果是偶发的零星点异常，无需大费周章修改全局参数，直接用鼠标点击中间可视化区域工具栏的 “橡皮擦” (Eraser) 工具，鼠标变成十字，拖曳在异常点上轻轻抹擦，即可将杂点直接置零，不参与后续计算。如果是大面积倍频，调小 Pitch Ceiling 能强行切除压制。
症状： 匹配出的词项名称与音频代表的读音完全不同。
急救： 检查音频文件命名。在模糊匹配下，系统会主动滤掉类似 001 等干扰项，但若命名仅写有 a.wav、b.wav，算法无法通过拼音与文字进行联系。在您能够百分之百保证文件导入队列排序与字表完全一行的前提下，请直接在匹配弹窗内选择 “顺序匹配 (Order Matching)”，可以绝防错。
症状： 原以为会获得大一统标注，结果切散成许多零星小文件夹或者只有单层信息。
急救： 检查您当前运行的工作流模式。在“批量音频”模式下，软件本就不会强行拼接出大 TextGrid，而是会对每个独立的 wav 音频在其目录下生成一个同名的 TextGrid。若需要大整轨 TextGrid，建议首先使用 Toolkit 进行音频缝合，进而采用长音频工作流提取。另外注意，在多发音人模式下，无法合并导出同一张 TextGrid 标注层。
急救： 当你双击 .teproj 唤醒软件时，PhonTracer 会认定你有明确的目标工程，因此会采用高优先级载入当前工程文件，并静默清理后台陈旧的自动保存工作区，防止旧缓存数据污染你的新项目。若要恢复上次死机前的数据，请直接双击打开主程序，不要双击外部工程包。
急救： 自动保存是“后台保命机制”，备份数据写在用户隐藏目录下，并非用于归档和数据分发的格式。要跨设备工作、数据备份或向他人交付成果时，请务必通过“导出项目 (.teproj)”生成无损打包文件。
急救： 这是新版的音节边界保护机制。应用 F0 参数或估计范围时，系统只会更新音高曲线，强制保护您已经辛苦拉好的红蓝边界线不被覆盖。如果您需要根据新参数强制重新切分，请在左侧面板点击“重新自动识别”或在 CLI 输入 recalculate，但这会覆盖你的手工修改边界。
急救： 共振峰高度敏感于发音人、元音类型和录音质量。请先点击参数面板上的“估计”按钮，把“系统推荐档”和“低上限复核档”都与声谱图共鸣带对照。成年男性通常可从 5000 Hz 左右起步，女性通常可从 5500 Hz 左右起步，儿童或较短声道发音人可能需要 8000 Hz 左右。分析上限过高或过低都可能导致 F1/F2 换轨；如果仅有个别跑飞点，再使用“剔除点”工具单点清除。
急救： 共振峰模式致力于母音（元音）空间特征分析，因此右侧的 F0 纵轴及蓝色音高曲线会被自动隐藏。若需要检查基频，请将“分析模式”一键切回“声调/F0”模式。
急救： 二维高斯核密度估计（2D KDE）是极度消耗 CPU 的算法。大样本项目建议关闭左侧面板底部的“实时刷新预览”，集中调整完配色、比例、分组与筛选后，手动点击一次“刷新预览”出图。
急救： 科学图表编辑器内部对当前项目数据和发音人树进行了高响应性的运行时缓存。如果允许在编辑器打开时导入新工程，很容易引起指针悬空导致崩溃。请关闭图表编辑器窗口，再进行导入操作。
急救： 为彻底规避 Windows + Tkinter 多进程架构下偶发的 GIL 锁多线程崩溃，主程序关闭了外部文件直接拖拽的 windnd 入口。请统一使用“添加音频”或“选择长音频”按钮，通过系统的文件资源管理器安全导入文件。列表内部的拖放排序仍可完美使用。
急救： .ptwl 的双击文件关联是由 Windows Setup 完整安装包注册的。如果你使用的是免安装的绿色 zip 压缩包版本，你的系统是不知道用什么程序打开它的。你需要先手动打开 Toolkit，然后再在里面选择载入您的 .ptwl 文件。
急救： 这些图需要极其特定的数据格式才能自动画出：组名必须能够直接解析成“前字调类+后字调类”（比如 阴平+去声 或者 T1+T4）。同时，你至少需要有 2 个前字调类、2 个后字调类和至少 4 种两两组合，并且必须有音节级别的 F0 采样！数据太少或者名字不对，系统都会自动隐藏该图以防产生误导结果。
急救： 不可以。自定义脚本并非万能的“系统操作台”，而是一个专为了解决重绘与排版需求的受控沙盒。禁止使用 open，禁止使用 pandas, seaborn 和 gaussian_kde（防卡死），它只能读取 ctx 对象中的底层科研数据并使用原生的 matplotlib 和限定库进行作图。
急救： 因为这是一个“协作式”停止，并不是直接暴力杀掉 Python 进程。你的脚本需要在大型循环中通过 ctx.is_cancelled() 自行检测停止信号。如果您交给 AI 写代码，AI 生成的代码中通常会包含这些安全退出机制。
