2026 年 6 月 1 日之后，PhonTracer 的新增功能主要集中在五条工作流：

## 三、Toolkit 自定义脚本：受控 AI 协作分析层

### 3.1 新增自定义脚本页

Toolkit 现在有独立的自定义脚本页。该页的核心区域包括：

- 左侧“脚本库”：列出本地保存的脚本，支持搜索和类型筛选。
- 脚本管理按钮：新建、编辑、删除、导入脚本、导出脚本。
- 右侧“运行与输出”：显示当前脚本名称、描述、源码预览。
- 运行控制：运行、停止、复制提示词、打开目录。
- 输出预览：显示脚本生成的图表，支持多图上一张 / 下一张。
- 运行日志：显示 `ctx.log()` 和 `print()` 捕获到的诊断信息。

手册建议写法：

> Toolkit 的“自定义脚本”页用于运行研究者或 AI 编写的受控图表脚本。它适合做常规图表工具尚未覆盖的探索性图表、参考图重绘、质量诊断和临时统计表，不建议用来替代主程序已经提供的稳定导出功能。

相关实现：

- `toolkit.py` 的 `build_script_tab()` 构建脚本页。
- `toolkit.py` 的 `on_new_script()`、`on_edit_script()`、`on_delete_script()`、`on_import_script()`、`on_export_script()` 负责脚本库操作。
- `modules/script_manager.py` 负责脚本 JSON 文件的保存、读取、复制、导入和导出。

### 3.2 脚本统一入口 `run(ctx)`

自定义脚本必须定义统一入口：

```python
def run(ctx):
    ...
```

`ctx` 是 `ScriptContext`，不是普通裸 Python 环境。脚本应从 `ctx.dataset.items` 或 `ctx.dataset.included_items()` 读取项目快照，而不是自己读取 `.teproj` 或本地文件。

脚本可以返回：

- 单个图表：`ctx.figure(fig, filename="xxx.png", title="中文标题")`
- 单个表格：`ctx.table(rows, columns, title="中文标题")`
- 图表和表格列表：`[ctx.figure(...), ctx.table(...)]`

可写入手册的最小示例：

```python
# 脚本名称：分组样本数统计
# 功能说明：统计每个字表组中纳入分析的条目数量

def run(ctx):
    counts = {}
    for item in ctx.dataset.included_items():
        group = item.get("group") or "未分组"
        counts[group] = counts.get(group, 0) + 1

    rows = [[group, count] for group, count in sorted(counts.items())]
    ctx.log(f"统计完成，共 {len(rows)} 个组。")
    return ctx.table(rows, ["组别", "纳入条目数"], title="分组样本数")
```

相关实现：

- `modules/script_api.py` 的 `ScriptContext`
- `modules/script_api.py` 的 `FigureResult` 和 `TableResult`
- `modules/script_api.py` 的 `DatasetSnapshot.included_items()`

### 3.3 脚本可用数据快照

脚本读取的数据来自 `.teproj` 中的 `project.json` 和缓存文件。`build_dataset_snapshot()` 会把项目转换成适合脚本读取的条目列表。

每个条目常用字段包括：

- `speaker_id`、`speaker_name`：发音人信息。
- `item_id`、`label`、`group`：条目 ID、标签和字表组。
- `wordlist_version`、`wordlist_title`：是否来自高级字表及其名称。
- `group_note`、`group_tags`：组备注和组标签。
- `item_note`、`item_tags`、`item_aliases`：词项备注、词项标签、别名。
- `item_meta`：高级字表中的自定义字段字典。
- `metadata_source`：元数据来源或复核状态。
- `is_excluded`：是否被排除，不参与默认分析。
- `analysis_mode`：`f0` 或 `formant`。
- `pitch`：原始 F0 轨迹，包括 `xs`、`freqs`、`t_values`。
- `syl_data`：音节级 11 点 F0 Hz 采样。
- `syl_t_values`：音节级 11 点五度标度。
- `formant`：原始共振峰轨迹，包括 `xs`、`f1`、`f2`。
- `syl_formants`：音节级 F1 / F2 采样。

手册必须提醒：

- 默认做论文图时，脚本应优先使用 `ctx.dataset.included_items()`，避免把已忽略条目混进分析。
- 跨发音人比较声调走势时，优先使用 `syl_t_values`，不要直接把不同发音人的原始 Hz 混在一起求均值。
- 结构类图表只有在 `item_meta` 中真的存在结构字段时才可靠。普通字表的括号分组不会自动变成 item 级结构元数据。

### 3.4 脚本运行限制和安全边界

自定义脚本不是任意 Python 运行环境。第一版运行器明确设置了边界：

- 允许导入：`math`、`statistics`、`collections`、`itertools`、`re`、`time`、`warnings`、`numpy`、`matplotlib`、`scipy`。
- 禁止导入：`os`、`sys`、`subprocess`、`requests`、`pandas`、`seaborn`、`plotly` 等未列入白名单的库。
- 禁止调用：`open`、`eval`、`exec`、`input`、`globals`、`locals`、`vars`、`compile`、`__import__` 等系统能力。
- 禁止高风险计算：`scipy.stats.gaussian_kde` / `gaussian_kde`。
- 图表后端使用 `Agg`，避免脚本自己弹出 Matplotlib 窗口。
- 脚本在后台线程运行，默认超时 30 秒。
- “停止”是协作式取消：脚本需要定期检查 `ctx.is_cancelled()` 才能尽快结束。

手册建议写法：

> 自定义脚本的目的不是让用户运行任意 Python，而是在 PhonTracer 提供的只读数据快照和安全边界内生成图表或表格。它更像一个受控研究沙箱：程序提供数据、接口和归档，AI 或研究者提供分析逻辑。

### 3.5 脚本输出目录和工程归档

脚本运行成功后，Toolkit 会保存输出：

- 图表保存到用户目录下的 `~/.phon_tracer/script_outputs/时间戳_脚本名/`。
- 图表格式允许 `.png`、`.jpg`、`.jpeg`、`.svg`、`.pdf`，不合法后缀会回退为 `.png`。
- 表格结果保存为 UTF-8 with BOM 的 `.csv`。
- 点击“打开目录”可直接查看输出文件。
- 点击图表预览区域可尝试用系统默认程序打开图像。

如果当前 Toolkit 已加载 `.teproj`，脚本成功运行后会询问是否写入工程记录。选择写入后，工程的 `project.json` 会增加 `custom_script_runs`，记录内容包括：

- `script_id`
- `script_name`
- `script_type`
- `api_version`
- `software_version`
- `code_sha256`
- `code`
- `used_at`
- `user_goal`
- `status`
- `outputs`

手册建议强调：

- 写入工程记录可以增强可复现性，适合论文数据归档。
- 不写入工程记录也不会删除已经生成的图表和表格，只是工程内部不保存这次脚本历史。
- 工程页提供“清理脚本运行记录”，只移除工程里的脚本历史，不删除本地脚本库、音频、条目、缓存或已导出的图片。

### 3.6 AI 脚本提示词生成器

Toolkit 中的“复制提示词”不只是简单模板。`modules/script_prompt.py` 会根据当前工程摘要和用户选项生成 AI 提示词，提醒 AI：

- 第一轮不要直接写代码，要先澄清研究目的。
- 推荐若干图表候选，并说明适合回答的问题、使用字段、统计口径和误用风险。
- 进入代码阶段后必须输出可直接放进 Toolkit 的 Python 代码。
- 脚本必须以两行元信息注释开头：
  - `# 脚本名称：...`
  - `# 功能说明：...`
- 脚本必须定义 `def run(ctx):`。
- 不读取本地文件，不访问网络，不使用未允许的库。
- 缺数据时返回空图或日志，不崩溃。

这里建议在正式手册中把它描述为“AI 协作入口”，而不是“内置脚本模板”。常规论文图仍应优先使用主程序图表工作台；自定义脚本更适合探索性分析、参考图重绘、临时统计和研究者尚未明确知道图名的图表发现。

## 四、高级字表 `.ptwl`

### 4.1 `.ptwl` 的定位

`.ptwl` 是 PhonTracer 的高级字表格式。它不是普通 `.txt` 字表的简单换壳，而是可以保存研究元数据的 JSON 文档。

普通字表一般只提供：

- 组名
- 词项 / 字词

高级字表额外提供：

- 字表名称
- 字表说明
- 组备注
- 组标签
- 词项备注
- 词项标签
- 别名
- 自定义字段
- 复核状态 / 元数据来源

相关实现：

- `modules/wordlist_v2.py` 的 `WORDLIST_SCHEMA = "phontracer.wordlist.v2"`
- `modules/wordlist_v2.py` 的 `create_empty_wordlist_document()`
- `modules/wordlist_v2.py` 的 `normalize_wordlist_document()`

### 4.2 Toolkit 高级字表编辑器

Toolkit 新增“字表”页，核心是 `VisualWordlistEditor`。界面大致分为：

- 顶部：字表名称、组数 / 项数 / 标签数 / 待复核数摘要。
- 左侧：组列表，可添加组、删除组、右键操作。
- 中间：词项表，显示词项、备注、标签、别名、补全状态。
- 右侧：属性编辑区，可编辑组和词项的详细元数据。
- 底部：打开 `.ptwl`、保存、全部标记为已复核、复制 Agent 生成字表提示词、同步到音频拆分。
- “更多”：新建字表、另存为、检查字表、导入 v1 文本、导入 CSV、导出 v1 文本、导出 CSV、删除当前组、删除当前词项。

可写入正式手册的操作流程：

1. 打开 Toolkit。
2. 进入“字表”页。
3. 点击“打开 .ptwl”载入已有高级字表，或直接新建。
4. 在左侧创建组，在中间添加词项。
5. 在右侧填写组备注、组标签、词项备注、词项标签、别名和自定义字段。
6. 如由 AI 自动生成元数据，可保留“AI推断，需人工复核”的状态；人工确认后点击“全部标记为已复核”。
7. 点击“保存”或“另存为”生成 `.ptwl`。
8. 如果要立刻用于拆分长音频，可点击“同步到音频拆分”。

### 4.3 `.ptwl` 与 v1 文本 / CSV 的互转

高级字表编辑器支持：

- 从普通 v1 文本导入。
- 从 CSV 导入。
- 导出为普通 v1 文本。
- 导出为 CSV。

CSV 导入时支持的常见列名包括：

- `group`、`组名`、`组别`、`group_name`
- `group_note`、`组备注`
- `group_tags`、`组tag`、`组标签`
- `label`、`词项`、`字词`、`word`、`item`
- `item_note`、`词项备注`、`备注`、`note`
- `tags`、`tag`、`标签`
- `aliases`、`别名`
- `pinyin`、`拼音`
- `tone`、`声调`
- `rhyme`、`韵母`
- `metadata_source`、`自动补全状态`、`review_status`、`复核状态`

除此之外，CSV 中未被消费的列会进入 `item_meta`。列名也可以使用 `meta.xxx` 的形式显式写入自定义字段。

手册建议说明：

> 如果你希望之后按“词频等级”“语义类”“音系结构”“实验条件”等变量作图或运行脚本，推荐在 CSV 中用自定义列保存这些信息，再导入为 `.ptwl`。普通 txt 字表无法可靠保存这些字段。

### 4.4 主程序导入 `.ptwl`

主程序导入字表窗口现在支持以下文件：

- `.txt`
- `.csv`
- `.ptwl`
- `.TextGrid`

当用户导入 `.ptwl` 时，程序会：

1. 读取高级字表 JSON。
2. 调用 `flatten_wordlist_document()` 生成普通分组列表和扁平词项。
3. 把高级字表转换成可显示的 v1 文本，显示在导入字表窗口中。
4. 保留原始高级字表文档，后续处理时传入长音频或批量音频工作流。
5. 对数量不匹配、重复标签、AI 推断未复核等情况执行 `validate_wordlist_document()` 检查。
6. 在生成每个 item 时调用 `apply_record_metadata()`，把组和词项元数据写入 item。

最终写入 item 的字段包括：

- `wordlist_version`
- `wordlist_title`
- `group_note`
- `group_tags`
- `item_note`
- `item_tags`
- `item_aliases`
- `item_meta`
- `metadata_source`

手册中应强调：

- `.ptwl` 可以像普通字表一样用于长音频切分和批量音频匹配。
- 但 `.ptwl` 的真正价值在于保留研究元数据，方便后续作图、脚本和归档。
- 如果用户在导入 `.ptwl` 后手动改了文本框内容，当前高级字表文档会被清空，后续按普通字表处理。

### 4.5 `.ptwl` 文件关联

Windows 安装器现在注册了 `.ptwl` 文件关联。安装后，用户可以双击 `.ptwl`，系统会用 Toolkit 打开：

```text
Toolkit.exe "%1"
```

相关实现位于 `installer.iss`：

- `WordlistAssocName = "PhonTracer Advanced Wordlist"`
- `WordlistAssocExt = ".ptwl"`
- `WordlistAssocKey = "PhonTracer.Wordlist"`
- 默认图标为 `{app}\_internal\assets\ptwl.ico`
- 打开命令为 `{app}\Toolkit.exe "%1"`

手册建议写法：

> `.ptwl` 是 PhonTracer 高级字表。安装 Windows setup 版本后，可以直接双击 `.ptwl` 进入 Toolkit 字表编辑器。若使用绿色压缩包版本，可能需要手动在 Toolkit 中打开。

## 五、图表工作台新增能力

### 5.1 图表分组规则

图表编辑器新增“图表分组”规则栏。它不会改变项目里的字表组，而是在绘图时临时决定“图上如何分组 / 配色 / 筛选”。

可选分组来源包括：

- 默认组：沿用字表组。
- 词项标签：按 `item_tags` 分组。
- 组标签：按 `group_tags` 分组。
- 自定义字段：按 `item_meta` 中选定字段分组。
- 复核状态：按 `metadata_source` 分组。
- 词项：按 `label` 分组。
- 发音人：按 `speaker_name` 分组。

标签类分组支持三种模式：

- 每个标签独立成组。
- 包含 / 不包含某标签。
- 仅筛选包含指定标签的条目，但仍按默认组绘图。

手册建议写法：

> 图表分组规则用于回答“按什么研究条件比较”的问题。默认组适合复现字表设计；词项标签和组标签适合探索目标词 / 填充词 / 变调 / 对照组；自定义字段适合正式论文中的实验条件、词频、音系结构或语义类别。

### 5.2 分组规则预览

分组规则弹窗会显示：

- 当前规则会生成多少组。
- 总共匹配多少项。
- 每组的样本数。
- 当组数过多时，提示论文主图建议筛选到 8 组以内。

这可以减少“导出后才发现分组错了”的情况。

手册可补充：

> 应用图表分组规则前，建议先看右侧“分组预览”。如果预览显示 0 组，说明当前规则没有匹配到可绘制条目，通常是 `.ptwl` 元数据缺失、标签拼写不一致，或选择了不存在的自定义字段。

### 5.3 二字组调类效应时间进程

声调图表新增“二字组调类效应时间进程”。该图用于二字组声调组合数据，估算：

- 前字调类影响
- 后字调类影响
- 交互项 / 残差

在两字组归一化时间轴上的占比变化。

使用条件：

- 当前选择的组别来自二字组调类组合。
- 组名需要能解析成“前字调类+后字调类”，例如 `阴平+去声`、`T1+T4`。
- 至少需要 2 个前字调类。
- 至少需要 2 个后字调类。
- 至少需要 4 个可用调类组合。
- 条目必须有两段音节级 F0 数据。

源码依据：

- `modules/acoustic_exporter.py` 的 `_split_tone_pair()`
- `_is_two_syllable_tone_entry()`
- `_has_tone_effect_time_data()`
- `_collect_tone_effect_time_rows()`
- `_plot_tone_effect_time()`

手册建议谨慎表述：

> 该图是探索性方差分解式可视化，用于观察二字组中前字、后字和交互项随时间的相对贡献。它不能自动替代统计建模结论，正式论文中仍应结合研究设计和统计检验解释。

### 5.4 调类组合前后字均值热图

声调组别概览图中新增“调类组合前后字均值热图”。当数据满足二字组组合条件时，概览指标中会出现该选项。

该图生成两张热图：

- 前字平均五度值
- 后字平均五度值

横轴是后字调类，纵轴是前字调类，单元格数值为平均五度值。

使用条件与“二字组调类效应时间进程”相同。若数据不足，图中会显示原因提示，而不是静默生成误导性图表。

手册建议写法：

> 如果研究目标是比较不同二字调类组合中前字和后字的平均调值，可使用“调类组合前后字均值热图”。导入字表时最好把组名设计为清晰的“前字调类+后字调类”，例如 `T2+T3`。

### 5.5 组别筛选器改进

图表编辑器的组别筛选体验有多项改进：

- 支持搜索组名。
- 支持按样本数排序。
- 支持全选当前可见组。
- 支持反选当前可见组。
- 支持自动选择高频组。
- 支持“论文主图”模式下自动精选样本量最大的 8 组。
- 支持“附录图册”模式下自动全选并切换 PDF。
- 支持从项目树当前选中的组或条目同步组别筛选。

手册建议补充一个操作建议：

> 论文主图优先控制在 8 组以内；如果需要展示所有组，建议使用“附录图册”意图并导出 PDF。组别很多时，预览区会分页显示，导出时会按分页生成完整图册。

### 5.6 图表分组和二字组图的边界

需要特别提醒：

- “图表分组规则”只影响绘图时的临时分组，不会改写项目树里的原始组名。
- “二字组调类效应时间进程”和“调类组合前后字均值热图”需要保留原始字表组，因为它们要从组名解析前后调类。
- 如果用户用自定义字段把二字组重新分组，二字组效应图仍会优先使用原始字表组，避免丢失 `前字+后字` 结构。

## 七、研究报告与 Excel 档案新增内容

### 7.1 项目文件指纹

研究方法报告现在会记录 `.teproj` 文件的 SHA-256 指纹。Excel 档案也会写入该校验码。

### 7.2 参数差异按多数值计算

“局部参数差异”现在以每个发音人纳入分析条目的多数值为基准，而不是以导出时界面最后停留的参数值为基准。

- 已忽略条目不参与多数值统计。
- 已忽略条目不进入参数差异清单。
- 多数值并列时，用发音人记录值兜底。
- Markdown 报告和 Excel 档案复用同一口径。

手册建议写法：

> 报告中的“局部参数偏离”表示某个条目与同发音人纳入分析条目的多数参数不一致。它不是和当前界面输入框比较，因此更适合用于检查研究数据内部是否存在个别参数例外。

- 组标签
- 词项备注
- 词项标签

- 元数据来源

这使 `.ptwl` 不只是前期导入工具，也成为后续研究归档的一部分。

手册建议提醒：

手册建议写法：

### 7.5 Toolkit 研究报告导出进度

Toolkit 的工程页导出研究方法报告时，现在有可见的阶段提示和加载状态，而不是长时间无响应。

手册建议只写用户可见行为：

## 八、CLI 新增自定义脚本命令

### 8.1 脚本库命令

CLI 新增以下自定义脚本相关命令：

- `list_scripts`：查看脚本库列表。
- `script_info <脚本ID或名称>`：查看脚本说明和源码。
- `script_save <脚本ID|new> <名称> <说明> <类型:chart> <代码文件路径>`：新建或保存脚本。
- `script_delete <脚本ID>`：删除脚本。
- `script_import <脚本JSON路径>`：导入脚本 JSON。
- `script_export <脚本ID> <目标JSON路径>`：导出脚本 JSON。
- `script_prompt [键=值 ...]`：生成 AI 自定义脚本提示词。
- `generate_prompt`：`script_prompt` 的别名。

手册建议在 CLI 章节增加示例：

```text
list_scripts
script_info "分组均值 F0 曲线"
script_import C:\Users\me\Desktop\my_script.json
script_export script_id C:\Users\me\Desktop\backup_script.json
```

### 8.2 `run_script`

`run_script` 可以按三种方式指定脚本：

- 脚本 ID。
- 脚本名称。
- 本地 `.py` 文件路径。

命令格式：

```text
run_script <脚本ID或名称或.py文件路径> [key=value ...]
```

支持参数：

- `timeout=30`：脚本超时时间，单位秒。
- `archive=true|false`：是否把脚本运行记录写入当前工程，默认 true。
- `desc="..."` 或 `user_goal="..."`：本次脚本用途说明。
- `name="..."`：自定义脚本名。
- `output_dir="..."` 或 `out_dir="..."`：指定输出目录。

示例：

```text
run_script "分组均值 F0 曲线" timeout=45 desc="论文图3：各声调五度值均值曲线"
run_script C:\Users\me\Desktop\custom_chart.py output_dir=C:\Users\me\Desktop\script_out archive=false
```

### 8.3 CLI 运行脚本的数据流

CLI 中运行脚本时，程序会：

1. 检查当前是否有项目数据。
2. 临时导出当前工程状态为 `.teproj`。
3. 用 `build_dataset_snapshot()` 构造脚本快照。
4. 调用 `run_custom_script()` 执行脚本。
5. 保存图表和表格输出。
6. 如果 `archive=true` 且当前有活动工程，把脚本记录加入当前工程。
7. 返回 JSON，包含日志、输出目录和输出文件列表。

手册建议强调：

> CLI 的 `run_script` 仍然走 PhonTracer 的受控脚本运行器，不会绕过安全检查。它适合自动化批处理和 AI Agent 操作，但不应被当作任意系统脚本执行器。

### 8.4 外部自动化仍被拦截

CLI 的 AI Agent 规则仍然强调：

- 优先使用内置 CLI 命令。
- 不要创建 `.bat`、`.cmd`、`.ps1`、额外 `.py` 等外部自动化脚本绕过 CLI。
- 每次只输入一个 PhonTracer CLI 内置命令，读完结果再继续。
- 出现 `success:false`、路径不匹配、音频和字表无法确认对应关系时应停下。

这与 `run_script` 不矛盾：`run_script` 是 PhonTracer 内置的受控脚本入口；外部脚本绕过 CLI 仍然不推荐。

## 十、建议新增到正式手册的完整操作流程

### 10.1 用高级字表管理实验材料

建议新增流程：

1. 打开 Toolkit。
2. 进入“字表”页。
3. 新建或打开 `.ptwl`。
4. 为每个组填写组名、组备注、组标签。
5. 为每个词项填写词项、备注、标签、别名。
6. 如需要结构化变量，点击“添加自定义字段”，例如 `结构=单字`、`词频=高`、`条件=变调`。
7. 如果元数据由 AI 生成，保留“AI推断，需人工复核”；人工确认后标记为已复核。
8. 保存 `.ptwl`。
9. 在主程序导入长音频或批量音频后，点击“导入字表”，选择 `.ptwl`。
10. 完成切分 / 匹配后，用右键“属性”核对条目元数据是否进入工程。

### 10.2 用标签或自定义字段生成论文图

建议新增流程：

1. 先确保条目来自 `.ptwl`，并有标签或自定义字段。
2. 打开图表编辑器。
3. 选择图表类型，例如声调轮廓图或元音共振峰空间图。
4. 在“图表分组”处点击“规则”。
5. 选择按词项标签、组标签或自定义字段分组。
6. 查看分组预览，确认每组样本数。
7. 如果组太多，使用标签筛选或论文主图模式控制到 8 组以内。
8. 导出 PNG / SVG / PDF。

### 10.3 用自定义脚本生成探索图

建议新增流程：

1. 在主程序中完成数据提取和人工复核。
2. 导出 `.teproj`。
3. 打开 Toolkit，进入“工程”页并载入 `.teproj`。
4. 进入“自定义脚本”页。
5. 点击“复制提示词”，把提示词交给 AI。
6. 让 AI 先澄清研究问题和图表方案，不要第一轮直接写代码。
7. 获得脚本后，在 Toolkit 中新建或编辑脚本，粘贴代码。
8. 点击“运行”。
9. 查看日志、预览图表和输出目录。
10. 如果结果用于论文或后续复现，选择写入工程记录。
11. 导出研究方法报告，让报告包含脚本源码和输出记录。

### 10.4 用 CLI 运行脚本

建议新增流程：

1. 启动 `PhonTracerCLI.exe`。
2. 用 `project_import <路径.teproj>` 导入工程，或按常规 CLI 流程处理音频。
3. 用 `list_scripts` 查看脚本库。
4. 用 `script_info <脚本ID或名称>` 检查脚本源码和说明。
5. 用 `run_script <脚本ID或名称> timeout=30 desc="用途说明"` 运行脚本。
6. 查看返回 JSON 中的 `logs`、`output_dir` 和 `outputs`。
7. 如需归档，保持 `archive=true`；如只是临时实验，可设 `archive=false`。

## 十一、建议新增到疑难解答的问题

### Q：`.ptwl` 和普通 `.txt` 字表有什么区别？

`.txt` 字表主要保存组名和词项，适合快速切分和匹配。`.ptwl` 是高级字表，可以保存组备注、组标签、词项备注、词项标签、别名、自定义字段和复核状态。后续图表分组、自定义脚本和研究报告都能使用这些元数据。

### Q：我导入 `.ptwl` 后，为什么文本框里看起来还是普通字表？

主程序会把 `.ptwl` 转换成普通字表文本用于显示和匹配，但内部仍保留高级字表文档。只要你没有手动改写文本框内容，导入处理时会把 `.ptwl` 元数据写入每个 item。

### Q：普通字表里的【组名】会不会自动变成结构元数据？

不会。普通字表的括号内容只是组名。若要让脚本或图表按“结构”“词频”“实验条件”等变量分组，应使用 `.ptwl` 的自定义字段，或从 CSV 导入这些字段。

### Q：自定义脚本可以读取我的本地文件吗？

不可以。第一版自定义脚本运行器禁止 `open`，也禁止导入 `os`、`sys`、`subprocess` 等系统库。脚本只能使用 PhonTracer 提供的只读工程快照。

### Q：为什么脚本里不能用 pandas / seaborn / plotly？

当前脚本运行器只开放了少量标准库、`numpy`、`matplotlib` 和 `scipy`。这是为了降低依赖和执行风险。若要画图，推荐使用 `ctx.plt` 或 `matplotlib`。

### Q：为什么 `gaussian_kde` 被禁止？

`gaussian_kde` 在数据点或网格较多时容易长时间占用后台线程。为了保护 Toolkit 稳定性，第一版运行器禁止该调用。需要密度效果时，可用散点、`hexbin`、`hist2d`、低分辨率分箱或抽样。

### Q：点击停止后脚本为什么没有立刻结束？

停止是协作式取消，不会强行杀死 Python 线程。脚本需要在长循环中定期调用 `ctx.is_cancelled()`，检测到取消后主动返回。

### Q：脚本运行记录写入工程有什么用？

写入后，`.teproj` 会保存脚本源码、代码 SHA-256、运行时间、软件版本、输出文件等信息。后续导出研究报告时，这些信息可以作为论文图表的可复现记录。

### Q：项目树右侧短标签看不懂怎么办？

右侧短标签只是快速提示。需要完整信息时，可右键条目选择“属性”，或查看异常提示、研究报告和 Excel 档案。

### Q：二字组调类效应图为什么不显示？

请确认当前数据满足条件：组名能解析为“前字调类+后字调类”，至少有 2 个前字调类、2 个后字调类、4 个组合，并且条目有两段音节级 F0 数据。

### Q：双击 `.ptwl` 没有打开 Toolkit 怎么办？

`.ptwl` 文件关联由 Windows setup 安装器注册。若使用绿色 zip 版本，系统可能没有文件关联，需要手动打开 Toolkit，再在字表页选择 `.ptwl`。

## 十三、这次补充中需要谨慎润色的点

1. 不要把自定义脚本写成“万能 AI 自动科研”。
   更准确的说法是：PhonTracer 提供结构化数据、运行接口、安全边界和归档链路，AI 或研究者在其中编写 `run(ctx)` 做探索性分析。

2. 不要把内置示例脚本写成主卖点。
   主程序图表工作台已经覆盖大多数常规论文图；自定义脚本的价值在于开放式探索、参考图重绘、临时统计和图表发现。

3. 不要承诺主程序能审查所有脚本方法学问题。
   程序能保存脚本源码、输出和代码指纹，也能限制危险系统能力，但统计口径是否合理仍需要研究者判断。

4. 不要把普通字表说成有完整元数据。
   普通字表只有组和词项；结构类、标签类和自定义字段分析应来自 `.ptwl` 或 CSV 元数据。

5. 不要把二字组调类效应图写成正式统计模型结论。
   它是探索性可视化，正式论文仍应结合研究设计和统计检验。

6. 不要把删除 `tests/` 目录写入用户手册正文。
   这属于维护性变化，不是最终用户功能。若需要记录，可放在内部开发说明中。
