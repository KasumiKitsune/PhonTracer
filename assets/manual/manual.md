# PhonTracer 使用手册

欢迎使用 PhonTracer！这不仅仅是一个“画一条音高曲线”的小工具，而是一个专门为语音学实验设计的全流程处理工作站。它的核心理念是将“录音、字表匹配、音节切分、基频数据采样、人工可视化修正、最终数据导出”这一整条枯燥的数据处理流水线，变得直观、可控且可复查。

无论你是习惯在图形界面中精雕细琢每一条声调曲线，还是希望借助代码和 AI 助手进行大批量的自动化提取，PhonTracer 都能为你提供得心应手的解决方案。

## 1. 核心概念与术语速查（读前必看）

在正式开始之前，我们先来理清几个在软件中频繁出现的概念。了解了它们，你会发现 PhonTracer 的运行逻辑其实非常符合人类的直觉。

> 💡 **术语小词典：**
> 
> * **基频（Fundamental Frequency, 简称 F0）：** 物理声学中反映声音高低的指标，单位是赫兹（Hz）。我们在语音学中常说的“声调的起伏”，在声学层面主要就是通过提取 F0 曲线来体现的。
> * **VAD（Voice Activity Detection，语音活动检测）：** 一种计算机算法。简单来说，就是让软件能够自动判断“录音里的这一秒钟是有人在说话，还是纯粹的环境底噪”。PhonTracer 处理长录音的第一步，就是靠它把大段录音“切萝卜”一样切成一个个带语音的片段。
> * **发音人（Speaker）：** 在实验中录音的具体对象。软件支持同时管理多个发音人，他们就像平行的工作区，各自的音频、修改进度和参数都不会互相干扰。
> * **词项（Item）与音节（Syllable）：** 软件处理数据的基本层级。一个“词项”对应你字表里的一行（比如一个词、一个短语或一句话）。软件拿到“词项”后，会根据规则把它进一步拆分成“音节”，并在每个“音节”的内部采集若干个 F0 数据点（比如默认采 11 个点）。
> * **TextGrid 格式：** 语音学界最常用的分析软件 Praat 所采用的标注文件格式。它里面记录了录音中哪一秒到哪一秒对应着什么字词。PhonTracer 支持无缝导入和导出这种格式。
> * **T 值（T-value）：** 一种用于消除发音人个体差异的数据归一化方法。举个例子，男低音和女高音读同一个调，绝对赫兹值天差地别。T 值通过对数转换，把所有人的音高都压缩映射到 0 到 5 的范围内（5 代表该发音人的最高音，0 代表最低音），这样不同人的声调走势就能放在同一张图里直接对比了。

## 2. 快速起步：选对入口，跑通流程

PhonTracer 并不是只有一个干巴巴的窗口，它其实包含了三个不同的入口，分别对应你在不同阶段的需求：

1. **软件主体（PhonTracer）：** 最核心的图形界面。适合绝大多数用户进行导入、参数调整、声谱图精修和数据图表导出。
2. **音频工具箱（AudioToolkit）：** 一个独立的音频前处理套件。如果你的录音材料太长需要拆分，或者太碎需要合并，在进主体软件之前先用它处理会事半功倍。
3. **命令行界面（PhonTracerCLI）：** 专为批处理、写脚本或是让 AI 编程工具（如 Codex、Trae）代劳而设计的纯文本交互界面。

### 2.1 软件启动方式与快捷入口

如果你下载的是打包好的 **Windows 安装包 (Setup 版本)**，这是最推荐的方式！安装包内置了 Inno Setup 简体中文界面引导，安装后，你的系统会获得以下便利：

* **全家桶快捷方式**：开始菜单里会自动生成 `PhonTracer 声调提取分析器`、`AudioToolkit 音频工具箱` 和 `PhonTracerCLI 命令行界面` 三个入口，随用随点。在安装过程中，您还可以自行勾选“创建桌面快捷方式”的可选项（默认不勾选），以便从桌面直接双击唤醒。
* **双击秒开工程**：安装包在系统注册表里注册了对 `.teproj` 工程文件的文件关联。以后做实验，直接双击你的 `.teproj` 文件，就能瞬间唤醒软件并恢复工作现场。
* **启动冲突解决**：当程序已经安装且存在旧的异常退出自动保存工作区时，若此时您双击打开一个新的 `.teproj`，PhonTracer 将优先尝试载入当前外部工程。外部工程只有在完整校验并成功替换工作区后才会生效；如果工程损坏或缺少资源，原自动保存工作区仍会保留，避免可恢复状态被提前清除。

如果你下载的是绿色免安装版 (zip 压缩包)，或者在 macOS 上使用 dmg 挂载，直接双击对应的 `.exe` 或 `.app` 程序图标即可。

> **快速穿梭传送门**
> 如果你已经打开了 PhonTracer 主程序，突然想用工具箱切个音频怎么办？不用去文件夹里翻找！直接点击上方菜单栏的**“关于”**窗口，里面提供了**“启动音频工具箱”**和**“启动命令行界面”**的传送门，一键无缝拉起。
> 
> 在底层运行机制上，关于窗口的传送门有着精细的安全保护设计：
> * **在源码开发环境中**：启动音频工具箱时会直接使用当前 Python 解释器去跑 `audio_toolkit.py`；当启动 CLI 时，系统会自动检查当前的 Python 环境。如果当前使用的是不具备控制台回显的后台进程解释器 `pythonw.exe`，主程序会自动将其替换为标准前台解释器 `python.exe`。这能确保在终端能够顺利新建控制台窗口并回显命令行界面。
> * **在已打包好的可执行文件环境中**：主程序会在同级目录下自动搜索 `AudioToolkit.exe` 并将其调起。而启动 CLI 时则会在目录下定位 `PhonTracerCLI.exe`，并强制为 Windows 系统新建独立的 CMD 控制台窗口，彻底避免命令行交互失效的问题。

对于有代码基础的开发者，在项目目录中安装好 Python 依赖后，可以通过终端呼出它们：

* 启动软件主体：`python main.py`
* 启动音频工具箱：`python audio_toolkit.py`
* 启动命令行界面：`python cli.py`

### 2.2 最小可行示例：一分钟跑通流程

如果你不想看长篇大论，只想知道这软件到底怎么跑起来，请跟着做一遍这个极其简单的流程：

1. 启动软件主体（运行 `python main.py`）。
2. 在左侧面板选择 **长音频模式**。
3. 把你自己的一段包含多词语朗读的录音（比如 `我的录音文件.wav`）拖入软件窗口导入。
4. 接着导入与之对应的纯文本字表（比如 `我的字表.txt`）。
5. 在右侧的树状列表中看看有没有标记为“需检查”的词项。如果有，点击它，在中间的声谱图中确认一下红色的起止边界对不对。
6. 确认没问题后，点击右下角的导出，选择 `xlsx` 表格。
7. 点击“导出项目（.teproj）”，把当前进度保存下来。

只要这个流程跑通了，就说明软件在你的电脑上运行完美，接下来就可以放心地处理海量数据了。

## 3. 软件主界面漫游指南

第一次打开 PhonTracer，你可能会被满屏的按钮和图表唬住。别担心，界面的布局逻辑极其清晰，它从左到右分为三个大区：“指挥中心”、“精修车间”和“成果仓库”。

### 3.1 左侧面板：全局指挥中心

这里掌管着软件的宏观运行状态与算法核心配置。

* **发音人管理：** 你可以添加发音人、切换当前发音人，或者开启“统一字表”功能。当多个发音人读取相同的实验材料时，开启“统一字表”可以一次导入、全局应用，避免重复导入字表的繁琐操作。
* **分析模式切换**：主面板参数区新增了**“分析模式”切换控件**。您可以在 **“声调/F0”** 和 **“共振峰/F1-F2”** 模式之间进行自由一键切换。切换后，参数调整区将自动折叠无关参数并展示对应模式的专属配置。为保障工作效率，PhonTracer 会在后台独立保存这两种模式下各个发音人与词项的边界状态，切换模式绝不会导致另一模式下已调整好的边界丢失。
* **导入模式切换：** 决定当前是处理一个长音频还是批量小音频。
* **全局参数折叠面板**：所有的算法核心参数（如基频上下限、共振峰数量、采样策略等）均收纳在此折叠区内。改动参数后，可以点击“重新自动识别”或“估计”来重新提取声学数据。

### 3.2 中间面板：可视化精修车间

当你需要像拿着放大镜一样检查某个词的具体表现时，中间面板就是你的主战场。
当你选中某个词项后，这里会显示该词的**声谱图**及其实时声学叠加线：
* 在 **F0 模式**下，声谱图上会叠加蓝色的 **F0 音高曲线**。
* 在 **共振峰模式**下，声谱图纵轴范围会自动适应最大共振峰频率，并在其上高亮叠加强调橘色（F1）和绿色（F2）的**共振峰轨迹散点**，同时右侧的 F0 纵轴会自动隐藏。

* **红线与蓝线：** 红线代表这个词的起止边界，蓝色的虚线代表词内部音节的交界线。发现自动切分不准时，可直接用鼠标拖动线条进行物理对齐微调。
* **橡皮擦与剔除点工具**：用于剔除声学提取中的噪声干扰点。
  * **在 F0 模式下**（按钮显示为“橡皮擦”）：鼠标在声谱图上呈现十字瞄准星，按住左键拖动可以像刷子一样将局部的 F0 音高点批量置零（归零点不参与后续的声声调曲线均值计算）。
  * **在共振峰模式下**（按钮显示为“剔除点”）：为防止鼠标滑动一笔抹消一整列宝贵的时间帧序列，该工具采用高精度点定位算法，**每次只会擦除距离鼠标中心最近的单个 F1 或 F2 点**。被剔除的点在后台会被记为无效的 NaN 占位。
  * **半径动态缩放**：在这两种模式下，您均可通过**鼠标滚轮**来实时放大或缩小十字橡皮擦的感应半径，极大提升了精细编辑效率。
* **多维播放控制与选区拖拽**：声谱图底部配备了全新的播放状态控制条，支持“播放”、“停止”、“自动”、“选区”、“当前字”、“整段”和“循环”等多种模式。
  * **鼠标拖选播放区间**：用户可在声谱图的非边界虚线区域直接按住鼠标左键并横向拖动，以在波形上拉出一个**淡黄色的播放选区**。
  * **高效听辨**：配合“循环播放”模式，系统会自动在该黄色选区内无限重播，特别适用于在微观分段处反复听辨边界是否包含多余爆破噪声，或确认共振峰突变点处的母音发音质量。

### 3.3 右侧面板：项目树与成果仓库

右侧面板主要用于数据的组织、排查和最终的导出。

* **分组与词项管理：** 这里以树状图的形式展示了你所有的字词。你可以双击重命名，或者拖动词项把它们归类到不同的分组里。
* **智能筛查机制与“需检查”视图：** “需检查”是一个动态的智能过滤器。
  * **声调模式下**：若某个音节内 F0 检出点比例过低，或疑似存在异常高低八度跳变，该词项会在此列表高亮出现。在声谱图上，被判定为疑似倍频/半频/噪声的异常频点也会以亮红色高亮警示。
  * **共振峰模式下**：若 F1/F2 发生错位（F2 小于等于 F1）帧比例偏高、有效帧数过少，词项同样会被归入“需检查”列表中。
* **右键忽略单项警告**：在项目树中，您可以右键点击任意标记警告的词项并选择**“忽略警告”**（也可随时“取消忽略”）。忽略后，该词项的异常检测将被绕过，不再呈现在“需检查”视图中。这适用于在人工确认为发音人特殊的清音现象或由于方言读音导致的数据缺失等合理特例。
* **一键导出与自定义警告拦截窗口**：所有分析好的数据均在此面板的一键导出菜单中生成。在点击导出时，如果工程中依然存在未处理的异常，系统会弹出一个**“可滚动自定义异常警告拦截窗口”**（`AnomalyWarningDialog`），直观罗列出所有的警告详情与数量，点击对应条目可快速定位至出错词项。您可以选择“返回修改”一键返回编辑，也可以确认无虞后点击“忽略警告并导出”强行完成输出。

## 4. 两大核心工作流与发音人管理

我们的录音材料千奇百怪，PhonTracer 为此设计了两套截然不同的工作流。但在介绍工作流之前，我们必须先弄懂“字表”是怎么一回事，因为它是贯穿一切的灵魂。

### 4.1 字表格式与音节拆分规则

字表可以是纯文本（`.txt`），也可以是从 TextGrid 里的 `words` 层转换过来的。最简单的字表就是一行写一个词。

**【示例 1】最简单的字表：**

```text
北京
上海
广州
```

**【示例 2】带分组标题的字表（方便后续按组统计）：**

```text
【普通话】
北京
上海

【英语】
bro/ther
sys/tem
```

*注：支持的分组标题形式包括 `【分组名】`、`[分组名]`、`# 分组名` 等。*

**【示例 3】一行写多个词（节约空间）：**

```text
北京 上海,广州，天津、南京
```

*注：你可以用空格、中英文逗号甚至顿号在同一行里隔开多个词。*

**音节拆分的玄机：**
软件拿到你的词后，是怎么知道它有几个音节的？它遵循以下智能规则：

1. **强行拆分符：** 如果你在词里写了斜杠 `/`，软件会无条件按照斜杠拆分。
2. **中文汉字：** 如果是纯中文，软件会自动把每个汉字当成一个音节拆开。
3. **外语单词：** 如果是连续的拉丁字母且没有斜杠，软件会把它当成一个整体（单音节）。

来看看直观的拆分效果表：

| 你在字表中写的文本    | 软件拆分出来的结果           |
|:------------ |:------------------- |
| `北京`         | `北`、`京`             |
| `bro/ther`   | `bro`、`ther`        |
| `computer`   | `computer` （作为一个整体） |
| `com/pu/ter` | `com`、`pu`、`ter`    |
| `天安/门`       | `天安`、`门`            |

> ⚠️ **注意：** 强烈建议你的字表里只写最纯净的文本标签，不要画蛇添足地写成 `001_北京`。因为如果你用了斜杠拆分（比如 `001_北京/大学`），软件会死板地把 `001_北京` 当成第一个音节，这显然不是你想要的。

### 4.2 长音频工作流（一段录音，包罗万象）

如果你让发音人坐在录音室里，一口气读完了几十上百个词，词与词之间有停顿，那这就是典型的长音频材料。

**操作步骤：**

1. 选择“长音频模式”，导入你的长录音文件（如 `.wav` 或 `.mp3`）。
2. 导入与录音顺序完全一致的字表。
3. 软件会开始狂奔：首先进行全段 VAD 语音活动检测，把一整段声音切成几十个小块；接着把字表里的词按顺序一个个扣在这些小块上；然后钻进每个词内部，寻找最响亮的元音区域；再切分内部音节；最后在每个音节里采点。

**可视化切分器：**
长音频由于容易受到咳嗽声、翻书声的干扰，自动切分可能会出现“多切了”或“少切了”的情况（比如检测出的语音段少于字表里的词数，软件会标注 `(缺失)`）。
此时你需要打开“可视化切分器”：

* **删除误检：** 对着被识别成语音的杂音段点右键，把它删掉。
* **手动框选：** 在空白的波形处点右键，可以手动新增一个片段。
* **调整边界：** 拖动红色竖线，确保片段正好包住语音。调整完确认后，软件会重新把字表按顺序对齐。

长音频模式还支持导入预先人工标注好的 **TextGrid**。如果你的 TextGrid 里有 `words` 层（词项边界）、`chars` 层（音节边界）甚至 `groups` 层，软件会优先提取这些极其精确的人工边界，完全跳过容易出错的自动 VAD 检测。

### 4.3 批量音频工作流（文件碎片化，化零为整）

如果你的录音已经被剪辑成了无数个独立的小文件（比如每个词对应一个音频），就需要用到批量模式。

**操作步骤：**
导入一批音频后，你可以直接使用，软件会把**文件名**当成词的名称，并提取整段音频的基频。但这种做法往往不够严谨，我们推荐在导入音频后，再导入一份字表，让软件将音频和字表“联姻”。

批量模式下导入字表有两种匹配逻辑，请根据你的情况对号入座：

| 匹配模式     | 适用场景                                           | 软件的处理行为                                              |
|:-------- |:---------------------------------------------- |:---------------------------------------------------- |
| **模糊匹配** | 文件名包含词项名称，但夹杂着各种编号或没用的后缀（比如 `001-北京(重录).wav`）。 | 软件会像侦探一样，忽略乱七八糟的字符，去寻找名字里包含“北京”的文件并对应起来。中文比汉字，英文比字母。 |
| **顺序匹配** | 文件名完全没有规律，但你在导入时文件的排序已经跟字表顺序一模一样了。             | 软件不再看名字，直接按排队的顺序，第一个音频对应字表第一个词，严丝合缝地扣上去。             |

在批量模式下，同样支持批量导入 TextGrid。软件会优先尝试找同名的 TextGrid，如果找不到，就按顺序兜底。

### 4.4 发音人管理与统一字表

研究往往需要采集多个发音人的数据。左侧的发音人列表就是你的“分身术”。
你可以不断添加新的发音人（比如“发音人A”、“发音人B”）。切换发音人时，那一瞬间，中间的声谱图、右侧的项目树、该发音人修改过的参数，都会像魔法一样恢复到他专属的状态。

如果你在做同一个实验，几十个发音人读的都是同一份字表，请务必在左侧开启 **“统一字表”**。这样你只要为第一个发音人导入过字表，后续再导入新发音人的音频时，软件就能自动套用字表和匹配规则，大幅提升效率。

## 5. 算法揭秘与参数调优（让你真正懂它）

要让提取出的数据精准可靠，不能只靠默认设置，你必须了解左侧那一排参数究竟在控制什么。让我们从算法原理出发，逐一击破。

### 5.1 宏观与微观切分的博弈

软件在找声音时，分为宏观和微观两步：

* **宏观（选地盘）：** 长音频的 VAD 阶段。主要通过计算声音强度的底噪，估算出一个动态阈值，把大块有声音的地方圈出来。决定“这个时间段对应哪个词”。
* **微观（找核心）：** 对每个词项寻找“元音核”。声调主要承载在元音和响辅音上，起头的爆破音（比如 p、t、k）通常是没有音高的杂音。软件会综合强度峰值、相对强度下降比例来锁定最值得采样的区域。

### 5.2 智能估测：告别盲猜 F0 范围

以前碰到新发音人，总得凭经验去猜基频上下限？现在大可不必！在 F0 参数区旁边，点击**“估计”**按钮，系统会火速扫描该发音人已导入的录音，摸清发音人的真实音高分布底戏。

其底层的物理与统计学估计逻辑如下：
1. **有声区间粗筛**：系统会首先以极宽的范围（如 50 Hz 到 700 Hz）作为基线对录音进行快速的音高探测，获得原始的有声帧序列。
2. **有声片段剪枝**：系统会自动识别并过滤掉时长过短的有声段（如短于 30 毫秒的零星浊音帧），以排除呼吸声或辅音残留导致的脉冲噪音。
3. **倍频抑制与跳变平滑**：为消除基频倍频（Octave Error）和半频跳变对整体统计分布的影响，算法会监测相邻帧之间的音高跳变比率。如果两帧之间的 F0 跳变幅度超过 20%（生理上人类声带振动很难在如此短时间内产生如此剧烈的音高突变），系统会将该连接处切断，单独进行片段权重过滤，使得最终估算出来的音高曲线分布更接近真实的声带基频。
4. **分位数统计分布计算**：在获得纯净的浊音基频点集后，系统会计算其统计分布，得出 P5、P10、P50（中位数）、P90、P95 等关键百分位数。
5. **生成三档范围建议**：
   * **保守范围 (Conservative)**：基于 P5 和 P95 百分位数，在两侧保留一定的冗余。此范围较宽，可以最大限度避免由于漏检导致的曲线断裂，适用于背景噪音较杂或发音人情绪起伏大、音域跨度极宽的场景。
   * **推荐范围 (Recommended)**：基于 P10 和 P90 百分位数，在准确性与信噪比之间达成最佳平衡。它能有效滤除大部分低频和高频的倍频杂音，适用于标准学术导出的声调分析。
   * **精细范围 (Fine)**：基于更窄的稳定区间，音高下限被设置得更高。这能强制压制发声末尾常见的气声或环境低频嗡嗡声，适用于高信噪比环境下的声谱精细绘制。

> **音节边界保护机制**
> 别担心应用这些“估计范围”会把你好不容易手动拽出来的红蓝边界线毁掉！当你点击应用推荐范围时，软件会采用**音高局部更新策略**：只刷新基频曲线和采样数据，而最大程度地保护你手工修改过的音节边界不受影响。

### 5.3 声调 (F0) 核心参数详解

调节以下参数，直接影响最终 F0 曲线的提取长相：

* **采样点数 (`pts`)：** 默认 **11**，范围 5 到 20。这表示在每个音节内部均匀采多少个 F0 点。传统的声调研究采 11 个点足以描绘出走势；如果你想做极其精细的微观变化研究，可以调大。
* **能量下降阈值 (`db`)：** 默认 **60**。它就像是切分微观元音核的一把尺子。数值越大，代表你越宽容，越容易把边缘比较微弱的声音（甚至是带点杂音的辅音）也框进提取范围；数值越小，条件越苛刻，边界切得越紧、越窄。如果你发现软件切得太窄，可以尝试增大它。
* **排除声母 (`skip_front`)：** 默认 **0**（秒）。这个参数的逻辑很精妙，它为“元音起点检测算法”设置一个搜索的盲区。如果你的材料爆破音特别重，适当把这个值调成 0.05 或 0.1，可以有效避免软件把开头的塞音算作有效音高。
* **Pitch Floor (最低基频)：** 默认 **75 Hz**。告诉软件检测音高的下限。如果是嗓音极其低沉的男低音，可以把它调到 60 甚至 50；
* **Pitch Ceiling (最高基频)：** 默认 **600 Hz**。检测音高的上限。儿童和女声可以适当提高；普通成年男声如果不小心喊破音或者出现倍频错误（曲线突然翻倍飙高），可以把这个值降低（比如降到 350）来强行压制倍频误检。
* **清浊判定阈值 (`voicing_threshold`)：** 默认 **0.25**。这是判定声音到底是不是“浊音（声带震动产生音高）”的标准。数值越低，软件就越容易把那些能量很低、带有气声或者严重嗓音化（creaky voice）的片段也判定为有声；数值越高则越保守，宁可测出 0 也不瞎猜。
* **修剪边缘静音 (`trim_silence`)：** 默认开启。在词的边缘把低于 -50 dB 的极弱声音剔除掉。
* **F0 提取算法：** 使用 Praat 经典的自相关 Pitch 方法。对于嘎裂声、严重不规则振动或弱周期信号，无法可靠测量时会保留为缺失值。不要为了得到连续曲线而强行补齐这些片段。

### 5.4 共振峰 (F1/F2) 分析模式大解密

当你在左侧把分析模式切换到**“共振峰/F1-F2”**，PhonTracer 就从“测调小能手”变身成了“元音探测仪”。

在语音学中，共振峰（Formants）是反映声道共振特性的核心指标。与代表声带振动频率的基频（F0）不同，共振峰主要是声道（咽腔、口腔、唇部）形状调谐的结果。在元音分析中，我们最关注前两个共振峰：
* **第一共振峰（F1）**：在声学上与**舌位的高低（口腔开闭度）**呈负相关。舌位越高（如闭元音 /i/ 或 /u/），口腔前部通道越窄，F1 的频率值就越低；舌位越低（如开元音 /a/），口腔开度最大，F1 频率值则会相应升高。
* **第二共振峰（F2）**：在声学上与**舌位的前后（前元音/后元音）**呈正相关。舌位越靠前（如前元音 /i/），舌体前推使前部声道容积减小，F2 的频率值就越高；舌位越靠后（如后元音 /u/ 或 /o/），舌根隆起使咽腔与口腔后部发生共鸣，F2 频率值就会显著降低。

利用 F1 与 F2 的这一声学特性，我们可以将 F2 作横轴（按反向坐标绘制）、F1 作纵轴（同样反向），绘制出直观反映人类发音时舌头在口腔中实际运动空间的**“元音舌位图”（元音空间图）**。

在共振峰模式下，PhonTracer 使用了经典的 **Praat Burg 法**（`praat_burg` 引擎）进行共振峰追踪。为了获取高精度的 F1 与 F2，用户需要细致调整以下几个参数：
* **追踪共振峰数量与最大频率**：默认追踪 5 个共振峰。在声道声学理论中，成年男性的声道较长（约 17 cm），其共振峰间隔较密，最大频率通常设为 **5000 Hz**；而成年女性与儿童的声道较短，共振峰间隔较宽，最大频率（`formant_max_hz`）通常设为 **5500 Hz** 甚至更高。如果设错，会导致共振峰点在声谱图上严重偏离真实的共鸣带。
* **分析窗长（`formant_window_length`）**：默认 **0.025 秒**（即 25 ms）。这在时间与频率分辨率之间提供了折中。较短的窗口可以捕捉更快速的语音瞬变，但在低频的分辨率上会打折扣；较长的窗口则会平滑掉微观的时间轨迹。
* **预加重起始频率（`formant_pre_emphasis`）**：默认 **50 Hz**。声带辐射会在高频处以每倍频程约 6dB 的速率衰减。预加重滤波器的作用是补偿高频衰减，抬高高频共振峰的能量，从而让高阶共振峰在算法中更容易被精准定位。
* **采样策略（`formant_sample_strategy`）**：
  * **整段11点**：在切分出的分析区间内进行等距 11 点采样。这对于研究双元音（如 /ai/, /ao/）的动态滑动路径，或是观察音节内部共振峰的协同发音（coarticulation）过渡过程非常有帮助。
  * **中段均值**：仅提取音节中间 1/3 长度段内所有共振峰帧的数学均值。由于元音在发音的中段往往达到最稳态的声道构型，这种策略能极大程度地避开音节起首与收尾处受前后辅音过渡（consonant transition）干扰的非稳态区域，非常适合用于绘制高一致性的单发音人元音舌位图。

**共振峰参数自动估计与抗错位机制**
共振峰分析的一大难点是 **F1/F2 发生“错位”（Swap Error）**，即在某些元音（如紧高元音 /i/）中，由于舌位极高且极靠前，低频的 F1 与高频的 F2 可能会因能量重叠，使得 F2 在图谱中错误地“下跌”至 F1 附近，或者 F1/F2 的分界线发生混乱。
为解决此问题，PhonTracer 内置了**共振峰自动推荐与评分机制**。点击估计按钮后，系统会抽样当前发音人的部分发音段，利用不同窗长和最大频率的组合运行 Burg 算法，并基于 F1/F2 的分离度、连续性与轨迹平滑度进行多维度评分，推荐三档实用方案：
1. **系统推荐档**：多项指标得分最高的均衡配置，适用于绝大多数常规语料的分析。
2. **抗错位档**：倾向于使用更长的分析窗长与更低的最大共振峰频率，通过拉大频率间隔、增加平滑度，最大程度防止 F1 与 F2 发生错位，尤其适合处理极端舌位元音或带有轻微底噪的录音。
3. **高分辨率档**：倾向于使用更短的窗长，不进行过多的时间域平滑，旨在忠实保留元音内部转瞬即逝的动态频谱结构。

## 6. 数据导出与实验记录

辛苦修好的数据，必须能够无缝对接到你后续的统计软件中。PhonTracer 提供了极其丰富的导出选项。

### 6.1 导出格式全解析

* **TXT（纯文本）：** 最基础的导出格式。使用标准制表符（Tab）进行列切分，不包含多余的警告信息，适合懂编程的用户直接拿用 Python、R 语言或 MATLAB 脚本快速读取底层数据。

* **XLSX（Excel 表格）[强推]：** 这是绝大多数实验人员的首选导出方式。PhonTracer 针对导出 Excel 进行了精密的学术实用化设计，文件在内部会自动生成两个独立的分析工作表：
  * **原始数据表**：完整罗列出所有词项在各个归一化采样点（如 11 点）上的原始基频数值点矩阵。
  * **分析结果表（带活公式）**：软件会根据当前发音人的基频最大值（maxHz）与最小值（minHz）作为统计常量，为每个数据点自动生成活体 Excel `T值` 归一化公式：
    `T = 5 * (log(Hz) - log(minHz)) / (log(maxHz) - log(minHz))`
    这意味着，当您在 Excel 中手动修正或剔除个别突变音高时，关联单元格的 T 值会自动根据公式实时重算，极大方便了数据的微调。

* **共振峰数据表（共振峰模式专属）：** 在“共振峰/F1-F2”分析模式下，导出的 `.xlsx` 文件会自动转换为主流的元音统计结构。除了输出各个音节在相应采样策略（整段11点或中段均值）下的 F1、F2 和 F3 的物理频值外，软件底层的 `write_formant_analysis_sheet_with_formulas()` 模块还会自动在 Excel 内新建一个独立的 `Formant Analysis` 工作表：
  * 该表采用 Excel 动态公式系统，自动按词语或组别检索数据行，并自动利用 `AVERAGE`、`STDEV` 等公式汇总各元音在 F1 和 F2 维度上的群体均值与标准差。
  * 所有的统计分析结果与原始数据通过活公式动态绑定。若您后续在 Excel 表的原始行中更正或剔下了个别跑飞的数据点，均值与离散度等声学指标会全自动智能重算，无需在外部统计软件中重新建模。

* **TextGrid 格式标注文件**：用于将已修正的精确边界导回 Praat 等专业声学软件中进行复查。PhonTracer 在导出 TextGrid 时遵循如下严密的学术规范：
  * **标准三层 IntervalTier 结构**：导出的 TextGrid 内部包含三个严格对齐的区间层：`groups` 层（组别信息）、`words` 层（词项/字表条目边界）以及 `chars` 层（仅在多音节词时自动生成，用于记录音节或单字的内部切分边界）。
  * **静音区间完整性填充**：为防止在 Praat 中打开时因存在“空白时段”导致的时间轴断裂，PhonTracer 会在有声波段之间的停顿、前白与尾白处自动填充标签为空的静音区间（Empty Interval），使得 TextGrid 从 `0.00` 秒至音频终点实现无缝平滑衔接。
  * **文件名 UUID 过滤优化**：在批量导出 TextGrid 时，系统内部的 `make_textgrid_export_stem()` 模块会精准剥离工程管理器为了防重冲突而自动为音频副本添加的 UUID 乱码及 `batch_数字_` 内部前缀，确保最终导出的每个 `.TextGrid` 文件名与您最初导入的原始音频文件名干干净净地一一对应，彻底解决数据集文件配对混乱的问题。

* **科学图表（声学可视化）[强推]：** 新版本中，PhonTracer 推出了全新的“声学科学可视化工具箱”，取代了旧版单一的折线图和 KDE 热力图。用户可以直接生成符合学术出版标准的声学图表，支持的格式包括 PNG、SVG、PDF 等。工具箱支持以下五大核心图表类型（详细说明请参见 **第 7 板块：声学科学可视化工具箱**）：
  
  1. **声调轮廓图 (contour)：** 展示各分组声调的动态平均曲线，支持叠加大底图个体细线或置信区间。
  
  2. **声调分布图 (distribution)：** 提供箱线图/小提琴图展示测量点离散分布、起中终三点对比、调域范围及变异系数(CV)。
  
  3. **时序密度图 (density)：** 使用连续核密度估计（KDE）计算在归一化时间轴上的声调概率密度热力分布。
  
  4. **数据质量检查图 (quality)：** 快速排查声带震动失效点（NaN）、异常噪点曲线，以及发音人调域中位数的异常八度跳变。
  
  5. **声调组别概览图 (overview_heatmap)：** 用矩阵热图的颜色深浅直观展现各声调分组在全局时序点上的均值或标准差。

### 6.2 导出编号规则与多发音人选项

导出数据时，软件会问你想要什么样的编号：
| 编号规则 | 含义 |
| :--- | :--- |
| **连续编号 (Continuous)** | 整个项目里所有的词项，不管在哪个分组，从 1 一直编号到最后（如 1, 2, 3... 100）。 |
| **组内编号 (Per Group)** | 重新编号！每个分组里的词项都从 1 开始算（如 A组 1,2,3；B组 1,2,3）。如果你需要严格对应原始材料里的实验编号，请谨慎选择。 |

如果你的项目里包含了多个发音人，在导出时会弹出额外的选项：

* **只导出当前发音人。**
* **分别导出全部发音人：** 为每个人生成一份独立的文件。
* **整合导出全部发音人：** 极其强大的功能！它会把所有发音人的数据糅合在一张大表或一张折线图中。不过请注意，TextGrid 格式不支持整合导出。

### 6.3 存档神技：工程包导入与自动保存

仅仅导出 Excel 和图片是不够的。在每日实验结束或阶段性工作完成前，建议手动导出 `.teproj` 工程包以归档。

**工程包的底层构成与机制**
`.teproj` 本质上是一个以 ZIP 格式压缩的标准化数据包。它并不只保存了一些轻量级的参数文本，而是完整封装了实验现场的全部要素。一个标准的工程包内至少包含：
* `project.json`：存储发音人元数据、项目结构、分段边界、人工修正状态、全局参数等结构化信息。
* `audio/`：工程工作区内独立存放的音频副本。
* `data/`：核心声学分析的缓存数据，其中基频（F0）和共振峰（Formant）数据均被保存为高压缩比的 `.npz` 二进制矩阵文件。

这意味着即使原始音频丢失或路径发生迁移，只要持有该工程包，便可在任何安装了 PhonTracer 的设备上无缝还原完整的实验状态。同时，共振峰等计算密集型分析在二次载入时会直接读取缓存，无需重新计算，大幅缩短了等待时间。

**工程导入预览与叠加冲突处理**
导入工程前，PhonTracer 会通过“工程导入预览窗口”（`ProjectImportDialog`）在后台解析工程包，并展示文件版本、发音人数量、已分析项目总数，以及每个发音人的音频工作模式（单条长音频或批量独立音频）与对应的项目计数。解析阶段会拒绝未知版本、非法归档路径、重复成员和超出限制的超大工程包，避免界面卡住或导入危险内容。

当当前工作区非空时，系统会要求用户选择以下两种冲突解决策略：
1. **覆盖导入**：清除当前所有工作区数据，完全被导入的新工程替换。
2. **叠加导入**：将导入工程中的所有发音人无缝合并到当前工程中。

在叠加导入过程中，为了防止数据覆盖并保证标识唯一，系统内部实现了一套严密的命名与 ID 重构逻辑：
* **同名发音人重命名**：如果导入的发音人名字（例如“发音人1”）与当前工程已有发音人重名，系统会自动为其追加递增后缀，改写为“发音人1_2”以示区分。
* **唯一标识符重构**：若两者的发音人 UUID（通用唯一识别码）发生冲突，系统会自动为新导入的发音人生成全新的 UUID，并将工程缓存数据（`.npz`）的路径及关联关系同步更新，确保多发音人树状结构数据的健壮性。
* **资源副本隔离**：导入资源会写入临时工作区并统一生成独立文件名。只有音频、缓存和状态全部校验通过后，系统才会原子替换正式工作区；任何失败都会回滚到导入前状态。

**自动保存的崩溃恢复与原子写入机制**
为了防止因系统死机、意外断电等不可抗力导致的数据丢失，PhonTracer 提供了完善的后台自动保存机制。其相关的配置与数据文件统一存储在用户主目录下的隐藏空间中：
* **数据存储路径**：`~/.phon_tracer/workspace`（工作区目录）
* **自动保存备份文件**：`~/.phon_tracer/auto_save_backup.teproj`
* **全局配置文件**：`~/.phon_tracer/config.json`

当用户在底部状态栏开启“自动保存”后，系统便会在每次发生项目变更时调用 `mark_modified()` 并触发后台保存逻辑。为保证性能与文件安全性，自动保存机制遵循如下步骤：
1. **延迟防抖**：修改操作发生后，系统会静置 2 秒，以防抖机制避免用户连续调整边界时产生高频写盘负担。
2. **定时轮询**：防抖期过后，后台写入定时器会以每 30 秒的频率周期性地持续同步数据。
3. **原子替换写入**：在写入磁盘时，为防止写入过程中程序意外终止导致原 JSON 损坏，软件会采用原子操作：先将数据写入临时文件 `project.json.tmp`，当确认数据完全落地且文件结构无损后，再调用操作系统的原子替换接口（如 `os.replace`）一次性覆盖正式的 `project.json`。
4. **启动自检恢复**：程序下次启动时，如果检测到工作区存在有效的自动保存数据，系统会判断其中是否包含有效的数据实体（如已分配的发音人、非空 items 等）。如果检测到数据有效，系统会弹窗询问用户是否恢复；如果仅是无意义的空工作区，或者检测到 JSON 文件本身已损坏，软件会静默清理该工作区，以防受损缓存阻塞程序启动。

**图表编辑器打开时的导入保护锁**
在导入工程时，系统会首先检测当前是否存在活动的“声学科学可视化工具箱”窗口。如果图表编辑器已打开，程序会强制锁定导入功能，并提示“图表编辑器已打开，修改图表期间禁止导入新工程”。这是因为图表编辑器内部对当前发音人进行了数据及参数缓存，如果在图表生成期间更改底层的 item 关系，会导致引用悬空或数据错位，该保护锁能有效规避由此引发的程序崩溃。

## 7. 声学科学可视化工具箱

为了满足语音学实验和学术论文发表对可视化图表的高标准要求，PhonTracer 对图表导出系统进行了全面重构，推出了**声学科学可视化工具箱**（Acoustic Scientific Visualization Toolbox）。

用户在右侧面板点击“导出 - 科学图表”或在 CLI 中使用 `export` 指令时，将激活该工具箱。以下对图表工作台的操控体验、声调与共振峰图表类型以及多维导出控制进行详细说明。

### 7.1 图表工作台：超爽的操作体验

新版的图表编辑器不仅仅是加了几个图，而是变成了一个专为语音学家设计的“图表工作台”。针对大项目渲染慢、调节卡顿等痛点，PhonTracer 从底层交互机制上进行了全方位的打磨：

* **后台异步渲染与一键取消**：为了避免大项目下或在大样本核密度计算中每调节一个滑块便导致界面卡死，所有的图表生成均交由独立的后台线程异步渲染。前台会显示加载进度条。如果您发现改错了参数，可以随时点击**“取消预览”**。当您快速连续调节多个控件时，系统会自动废弃前几次的后台计算任务，只保留最后一次配置进行渲染，操作响应极其平滑。
* **实时刷新与手动刷新**：折叠栏底部配备了“刷新预览”按钮及“实时刷新”开关。大样本量或需要启用高精度二维 KDE 的项目，建议关闭“实时刷新”，集中修改完图表类型、分组、筛选和比例等参数后，一次性手动点击“刷新预览”，以减少后台不必要的计算开销。
* **尺寸比例与 DPI 自适应高清导出**：受够了千篇一律的方形图？在“图像尺寸与比例”面板中，系统提供了默认、4:3、16:9、3:2、1:1 等常用纵横比，并支持滑块自定义比例。在导出 PNG 等位图时，通过设定“PNG 最小边像素”参数（如 1080 px、2160 px），软件会自动换算对应的 DPI 参数，确保导出的图表在大版面上依旧字迹清晰、无马赛克（学术发表仍首推导出 SVG 或 PDF 矢量图）。
* **悬浮模式 (Floating Mode)**：点击“悬浮模式”后，编辑器将自动折叠繁杂的左侧设置栏，精简底部底栏，窗口置顶显示为一个约 450x380 像素的迷你小窗。这能方便您一边在中间声谱图面板用橡皮擦修剪点，一边在置顶小窗内实时观察格局图的形态变化。
* **键盘快捷翻页**：在预览多受试者分别导出的图册或组别分页图册时，无需使用鼠标频繁点击，**直接按键盘上的左、右方向键**即可光速进行前后翻页。
* **图表缓存失效机制**：当您在主界面修改边界、剔除异常频点或切换当前发音人后，图表编辑器内部会自动侦测并清空已缓存的声学矩阵数据（`invalidate_cache()`），确保预览图表绝对反映最新的修正状态。

### 7.2 声调 (F0) 专属图表家族

若您的分析模式处于“声调/F0”模式下，工具箱将为您提供五把格局分析利器：

1. **声调轮廓图 (Tone Contour Chart, `contour`)**：即经典的**声学格局图**。
   * **横轴展现形式 (`contour_x`)**：
     * **归一化采样点 (normalized)**：横轴使用等距的采样点（如默认的 11 点/22 点时序展开），多个音节之间会绘制灰色虚线进行区隔，最适合比较不同声调的时序走势。
     * **真实物理时长 (duration)**：横轴使用以秒(s)为单位的真实物理时长。软件会计算每个音节的平均时长，并将其沿横轴累加展现，能够同时反映音高走势和时长差异。
   * **曲线展示要素 (`contour_content`)**：
     * **仅组别平均曲线 (average)**：只显示各组声调的平均走势曲线，画面清爽。
     * **平均曲线 + 个体浅色细线 (average_individual)**：在粗体均值曲线下方，用同色系的浅色半透明细线绘制出每一个发音样本 of 的原始轨迹，有助于直观观察组内变异。
     * **平均曲线 + 置信区间阴影 (average_sd_ci)**：在均值曲线两侧填充标准差（SD）或置信区间的半透明阴影，适合科学展示数据的离散程度。
   * **分面子图排版 (`contour_facet`/`facet`)**：支持单图重叠对比、按声调类型分面、按多音节词音节位置分面等。
2. **声调分布图 (Tone Distribution Chart, `distribution`)**：用于定量分析声调基频数据在特定位置或整体区间上的分布特征与变异性。支持绘制**科学箱线图**或**小提琴图**，用以展现采样点精细分布、起中终三点比较、调域范围跨度，以及组内发音一致性的变异系数（CV）比较。
3. **时序密度图 (Pitch Time Density Chart, `density`)**：利用二维高斯核密度估计（2D Gaussian KDE），以相对时间为横轴、音高为纵轴，绘制声调运动的时序密度热力图。
   * **截断极值处理 (`density_f0_mode`)**：二维高斯密度计算极易受边缘离散噪点影响。工具箱提供了 **5%-95% 分位数自动截断**、Min-Max 极值范围以及手动指定频率过滤等三种模式，以排除杂音干扰，使密度云团集中呈现发音人主流发音音强音域。
   * **归一化范围**：支持全局归一化（Global）和按受试发音人独立归一化（Speaker）。
4. **数据质量检查图 (Data Quality Check, `quality`)**：导出数据前进行“全局体检”的排雷利器。
   * **逐项基频曲线质量分布叠加 (raw_overlay)**：将所有词项的基频曲线叠加在一起。系统会通过算法智能检测质量异常的音频，将带有“致命错误（如数据全为NaN/0）”或“警告（有大段数据空洞）”的曲线以**红色虚线**高亮标出，正常曲线以**蓝色细线**绘制，方便一目了然定位问题曲线。
   * **有效基频检出比例分布比较 (active_ratio)**：以发音人为横轴绘制有效检出率的箱线图。图中含有一条红色虚线表示的 **60% 检出率警戒线**。如果某个发音人的中位数低于此线，说明其录音存在严重的清音/无声异常，需要重新测音高或重录。
   * **发音人基频中位数及调域离散域 (speaker_means)**：绘制每个发音人各个发音词项的基频均值散点，并用黑点和误差棒标记其调域中位数及四分位区间。可用于极其直观地排查是否有由于算法引擎错误导致的高八度（倍频）或低八度（半频）异常跳变点。
5. **声调组别概览图 (Tone Group Overview Heatmap, `overview_heatmap`)**：用大矩阵热图的方式呈现大样本均值（RdYlBu_r 蓝红渐变）或标准差（Reds 红色渐变），方便快速扫视大批量词项的时序离散程度。

### 7.3 共振峰 (F1/F2) 专属图表家族

若在主界面或参数面板将分析模式切换为“共振峰/F1-F2”，科学工具箱将自动装载共振峰绘图功能：

1. **元音共振峰空间图 (Vowel Space Chart, `formant_space`)**：**语音学研究的最核心标配图表！**
   * **声学坐标轴反向惯例（Academic Inversion）**：图表横轴为 F2 频率，纵轴为 F1 频率。根据语音物理声学的基本原理，**横轴 F2 越往右数值越低（反向），纵轴 F1 越往下数值越高（反向）**。这种坐标轴设计使得绘制出来的图谱分布方向与发音人在口腔中的实际舌位分布完全吻合（左上角为舌位高、前的前元音 /i/，右上角为舌位高、后的后元音 /u/，下部为舌位低、大开的开元音 /a/），直观还原了物理发音器官在口腔中的运动空间。
   * **丰富展现要素**：支持按发音人、词项或分组对元音散点染色；支持一键生成 **50% 或 95% 置信椭圆 (Confidence Ellipse)** 以科学评估元音音域分布集中度与变异域；支持叠加强调时间渐变轨迹连线，清晰呈现单个元音在音节内部随时间演变而发生的声学滑动走势。
2. **共振峰时序轨迹图 (Formant Trajectory, `formant_trajectory`)**：在归一化时间轴上分轨追踪 F1、F2 和 F3 的曲线演变。对于研究双元音（如 ai, ao）的过渡路径，或是观察音节前部与尾部的协同发音过渡（coarticulation transition）具有无可替代的作用。
3. **共振峰密度分布图 (Formant Density KDE Map, `formant_density`)**：当元音散点数量达数千个时，普通的散点元音空间图会重叠成墨黑色的一团。该图表利用二维高斯核密度估计（2D KDE）呈现概率分布密度，以漂亮的“热力星系云团”科学地勾勒出元音空间的核心聚合带。
4. **共振峰组别概览图 (Formant Group Overview, `formant_overview_heatmap`)**：以矩阵热图形式快速扫视所有词项或分组在 F1/F2 上的稳态均值与离散标准差表现，常用于大样本的附录画册。

### 7.4 分页图册与多组别筛选器

图表太多塞在一张图里像一团乱麻？PhonTracer 为此设计了智能组别筛选与自动分页系统：

* **多组别分页策略**：当项目中的分析分组（如声调类型或词项数）多于 8 个时，为避免线条堆叠导致视觉拥挤，系统在导出非热图类图表时，会根据目标格式自动分页：导出为 `.pdf` 时会整合生成为一本**多页 PDF 图册**；导出为 `.png`/`.jpg` 时，则会在目标目录下自动渲染生成以 `_第1页`、`_第2页` 为后缀的**分页图像序列集**。
* **组别优化建议**：对于组别数量过多的项目，编辑器会贴心弹窗提供三种展示策略：
  * *精选对比*：只筛选几个核心组别放在同一张图表进行严谨对比。
  * *批量图册*：自动将所有组别分页渲染，最终生成画册。
  * *整体概览*：忽略细节线条，直接生成组别概览热图以进行大尺度趋势扫视。
* **智能组别筛选器**：支持实时输入拼音/汉字搜索过滤分组、一键全选/反选；支持按各组有效发音样本量（Sample Size）进行升降序重排，并支持输入样本量阈值过滤掉样本量过低的不合格分组。
* **滚动穿透冲突优化**：新版本中，特别优化了筛选器内部的滚动机制。当鼠标悬停在筛选器长列表内滚动滚轮时，仅滚动内部列表，不会带动外层图表编辑器长设置面板一起晃动，彻底告别内外滚动冲突。

### 7.5 多发音人导出范围控制

在导出科学图表时，工具箱支持对项目中多个发音人的数据进行交叉选择与整合：

* **active (当前发音人)**：仅生成并导出当前选中的单发音人科学图表。
* **separate (分别导出)**：依次遍历发音人树，为工程中的每位发音人分别生成其专属的独立图表文件。
* **integrated (整合导出)**：**极其强大的受试合并功能！** 软件会自动将所有受试发音人的声学物理数据合并，进行统一的受试者归一化（Speaker Normalization）对数处理，从而生成代表该实验群体平均声学格局的均值格局图或核密度概率图，极大地方便了集体对比分析。

## 8. 进阶利器：AudioToolkit 独立处理套件

有时我们收到的录音极其混乱：要么是一段长达一小时却没有字表的乱弹，要么是几百个命名乱七八糟的微信语音。此时，建议你在进入 PhonTracer 主体前，先开启 `AudioToolkit` 独立音频处理套件给材料“洗个澡”。

### 8.1 合并音频（碎片缝合）

如果你想用长音频模式的连贯工作流，但手头全是一堆碎音频，用它！

1. 把多个 `.wav` 或 `.mp3` 拖进去。
2. 你可以随意拖动它们调整排队顺序。
3. 设定文件间的“静音间隔”（默认 0.5 秒，这能帮助 VAD 算法极好地认出边界）。
4. 点击合并。软件不仅会把它们无缝拼接，还会自动统一重采样到标准的 44100 Hz，输出一个纯净无暇的长 WAV 文件。

如果你有一份字表，这里还有一个绝活：**“导入字表自动排序”**。只要音频文件名里包含了字表里的词，软件就会自动把文件名跟字表做模糊匹配，瞬间把文件按字表顺序排得整整齐齐，没匹配上的会自动扔到最后面。

### 8.2 拆分长音频（一键切块）

假设你在做语料库整理，要把一长段录音切成几百个带标号的小文件。

1. 载入你的长录音文件。
2. 粘贴或导入字表（一行一个词）。文件解析内置了自动剥离文件开头 UTF-8 BOM 字符的逻辑，防止因可见/不可见 BOM 字符导致首行匹配失效。
3. 点击“一键匹配”，软件会自动进行宏观的语音活动检测（VAD）。如果发现切得不对，可以进入段落编辑器，利用鼠标滚轮进行波形图横向滚动，或使用 Ctrl + 滚轮对波形进行水平缩放，精细调整边界、点击播放听辨、删除误检段或添加手动段落。
4. 设定边缘缓冲时间（比如默认留 0.1 秒的白头白尾，听起来更自然）。
5. 点击拆分，你会瞬间得到一个目录，里面的文件被完美命名为 `001_北京.wav`、`002_上海.wav`... 文件名里如果存在系统不允许的非法字符，软件会自动替换为合法下划线，绝不报错卡死。

### 8.3 工程包结构预览与另存

AudioToolkit 中内置了高效的工程操作模块，专为无需载入主程序的大型工程分析而设计：
* **结构预览**：载入 `.teproj` 工程包后，工具箱会以极轻量级的方式解析其内部结构，快速呈现工程版本、发音人名册、各发音人工作模式（长音频/批量）、音频数量和已分析项目总量。这为数据交付与多人合作工程审查提供了极大便利。
* **工程包转换另存**：支持将打包好的 `.teproj` 另存为标准的 `.zip` 压缩包，以便于研究人员直接解包并读取底层的 `project.json` 结构或提取缓存在里面的音频副本及声学缓存。

### 8.4 界面导入与拖拽稳定性说明

在 Windows + Tkinter 的特定多线程运行环境下，直接使用窗口级底层鼠标拖放（如 `windnd` 库）极易与 Python 的 GIL（全局解释器锁）发生冲突，甚至在导入大规模无序文件时导致程序崩溃。
为了保证实验软件的高度稳定性，在新版 AudioToolkit 中：
* **稳定路径导入**：关闭了窗口级的外部文件直接拖入入口。推荐用户使用界面中的“添加音频”、“选择长音频”或“选择工程文件”按钮，通过标准文件浏览器对话框进行文件导入。
* **列表拖动排序**：导入列表后，其内部的拖拽排序、Delete 键删除选中项等交互机制依然被完美保留且稳定运行。

## 9. 批处理与 AI 助手：CLI 命令行完全指南

随着大语言模型（LLM）的爆发，越来越多人习惯用类似 Trae、Cursor、Codex 这样的 AI 编程助手来干活。为了满足极客用户和 AI 自动化的需求，PhonTracer 提供了强大的纯文本命令行界面（CLI）。

它能干几乎所有图形界面能干的事，并且所有的输出都是结构化的 JSON 格式，AI 工具读起来简直如鱼得水。

### 9.1 启动方式与基本语法

如果你是源码运行，在终端输入 `python cli.py` 即可进入交互模式；Windows 打包版则是双击运行 `PhonTracerCLI.exe`。

PhonTracer 的命令行不仅支持高度交互的 Shell 模式，还对非交互式及 AI Agent 自动化进行了精细优化：
* **单指令单次执行**：支持如 `PhonTracerCLI.exe status`、`PhonTracerCLI.exe --help` 或 `python cli.py status` 等启动形式。当启动参数不是以 `-` 开头的未知选项时，CLI 会将其直接视作单条内置命令运行，并在标准输出上返回格式化数据后安全退出。
* **多进程与 Agent 启动容错**：Windows 打包版在底层调用了 `multiprocessing.freeze_support()` 并在参数解析时过滤忽略了以 `--multiprocessing-*` 开头的系统级内部启动参数。这有效规避了在 Windows 系统下进行多线程/多进程基频重算时，子进程启动被误识别为用户命令输入的死循环异常。

你可以随时输入 `help` 查看所有可用指令，输入 `agent_guide` 获取 AI 操作守则 JSON，或者输入 `status` 查看当前发音人和声学参数状态。

### 9.2 核心指令词典

命令行中所有指令的参数均可用空格隔开，路径参数若包含空格请使用双引号包裹：

* **全局设置与声学参数配置：**
  * `lang zh` / `lang en`：切换输出语言为中文/英文。
  * `speakers`：列出当前项目所有的发音人。
  * `add_speaker 发音人2`：新建一个名为“发音人2”的发音人平行工作区。
  * `set_params pts=11 db=60 pitch_floor=75`：配置声学基频提取参数链。
  * `set_params analysis_mode=formant formant_max_hz=5500 formant_count=5 formant_window_length=0.025 formant_pre_emphasis=50 formant_sample_strategy=整段11点`：**【共振峰专属】** 配置共振峰模式下的精细分析参数。

* **工程导入与导出类指令：**
  * `project_import "我的项目.teproj"`：载入完整工程包。
  * `project_export "保存的项目.teproj"`：**【核心交付指令】** 将发音人配置、音频副本、边界状态及声学 `.npz` 矩阵完整打包为标准化 `.teproj`。换电脑、数据归档或交付给他人时，必须使用此命令导出，普通的 `project_save` 仅在本地内部工作区同步，不可用于外部交付。
  * `autosave`：读取当前自动保存的配置状态。
  * `autosave on` / `autosave off` / `autosave now`：控制自动保存功能的启闭，或强制后台立即进行一次原子替换写入备份。

* **有声数据提取与边界保护：**
  * `load_long "我的录音/长录音.wav"`：导入长音频。
  * `load_batch "文件1.wav" "文件2.wav"`：导入多个小音频。
  * `apply_wordlist "字表.txt" fuzzy`：应用字表。后面的 `fuzzy` 代表模糊匹配；`order` 代表顺序对齐。
  * `detect_f0`：不带参数时，估测发音人基频分布，以 JSON 格式返回 P5 到 P95 分位数建议。
  * `detect_f0 recommended` / `conservative` / `fine`：一键应用对应基频区间，并采用音高局部更新策略重算数据，**强制保护您手工修剪过的音节边界红蓝线不被重置**。

* **AudioToolkit 与数据集批量转化指令**：
  * `tool_merge "输出.wav" 0.5 "小音频1.wav" "小音频2.wav"`：命令行快速音频缝合。
  * `tool_split "长音频.wav" "字表.txt" "输出目录" 0.1`：一键拆分长音频。
  * `tool_sort_batch "字表.txt" "音频1.wav" "音频2.wav"`：批量音频模糊排序。
  * `import_batch_and_export <folder_path> <output.teproj>`：**【大样本转换神器】** 扫描指定文件夹（支持子目录按发音人命名归类），自动收集并成对匹配其中的 `.wav` 音频与 `.TextGrid` 标注文件（支持同名、包含和自然自然排序兜底），以 TextGrid 边界为基准进行批量声学分析，直接合成为一个标准化的 `.teproj` 工程包，极大地方便了大规模已有标注数据集向 PhonTracer 的迁移。

* **检查、微调与图表导出指令：**
  * `list_items warnings`：让系统把那些提取出 0 或者报错的词全部列出来（相当于界面上的“需检查”筛选）。
  * `modify_bounds 3 1.250 1.820`：极度精确的微调！把编号为 3 的那个词，起止时间强行改为 1.250 秒到 1.820 秒。
  * `recalculate`：根据当前设置的参数全局重新计算。注意：这会覆盖掉你之前手工调过的边界，请谨慎使用！
  * `export <格式> <输出路径> [编号规则] [目标] [参数=值 ...]`：以高度自由的参数定制导出数据。
    * *Excel 导出示例*：`export xlsx "分析结果.xlsx" continuous active`（导出当前发音人的基频数据且 T 值活公式自适应）。
    * *元音空间图导出示例*：`export formant_space "元音舌位.svg" active groupby=group`（以矢量图格式导出元音空间椭圆分布图）。
    * *时序密度图导出示例*：`export density "时序密度.png" integrated normalization=speaker facet=group`（合并发音人、按组别分面导出的声调密度热力图）。

### 9.3 管道与批处理文件的终极魔法

CLI 最强大的地方在于它可以被自动化脚本“操控”。你可以把一大串命令写在一个文本文件（比如 `commands.txt`）里，或者通过 PowerShell 的管道一次性喂给它。

**【示例 4】长音频自动提取的 PowerShell 批处理脚本：**

```powershell
@'
lang zh
load_long "我的录音/多语言.wav"
apply_wordlist "我的字表/多语言.txt"
export txt "自动化测试/long_results.txt" active
export xlsx "自动化测试/long_results.xlsx" per_group active
export textgrid "自动化测试/long_results.TextGrid"
project_export "自动化测试/保存的项目.teproj"
exit
'@ | python cli.py
```

**【示例 5】批量音频自动提取的 PowerShell 批处理脚本：**

```powershell
@'
lang zh
load_batch "散音频/001_北京.wav" "散音频/002_上海.wav" "散音频/003_广州.wav"
apply_wordlist "我的字表/字表.txt" fuzzy
export xlsx "自动化测试/batch_results.xlsx" continuous active
exit
'@ | python cli.py
```

### 9.4 给 AI 助手的专属提示词（Prompt）

如果你想让 Trae 或者 Codex 帮你代劳，不要让它们去直接修改源代码或者底层文件，这很容易搞坏系统。请直接把下面这段话喂给 AI：

> **“你不是来写自动化脚本的。你只能作为 PhonTracer CLI 操作者工作。**
>
> **请在当前项目目录中使用 PhonTracer CLI 完成任务。优先运行打包后的 PhonTracerCLI.exe；只有在明确处于源码开发环境且没有 exe 可用时，才使用 python cli.py。**
>
> **必须使用交互式 PhonTracer CLI，一步一步完成任务。每次只能输入一个 PhonTracer CLI 内置命令；必须等待并阅读该命令返回结果后，才能决定下一条命令。**
>
> **严禁创建或运行任何额外脚本文件，包括 .bat、.cmd、.ps1、.py、.r、.praat、.js、.vbs。禁止预先生成命令清单，禁止批量灌入多条命令，禁止用外部脚本处理音频、工程文件、导出文件或路径问题。**
>
> **所有音频导入、标注、边界修正、参数调整、质量检查、工程保存和结果导出，必须通过 PhonTracer CLI 的内置命令完成。需要批量音频导入时，只能在交互式 CLI 中调用 PhonTracer 自带的 load_batch 等命令，不能用外部脚本替代。**
>
> **如果 CLI 返回 success:false、出现警告、路径不匹配、音频和字表无法确认对应关系，或当前 AI 工具无法稳定交互运行 CLI，必须立即停止并向用户说明情况，等待用户确认。不要继续猜测，不要写脚本绕过，不要复制、改名、移动原始音频来规避问题。**
>
> **最终必须先导出完整 .teproj 工程包，确保可复现；然后再按用户要求导出 xlsx、TextGrid、wav、图表或其他结果文件。**
>
> **完成后只汇报：刚才逐步执行了哪些 CLI 命令、每一步是否成功、导出了哪些文件、是否有警告或失败项、下一步需要用户确认什么。”**

通过这段提示词，AI 就能化身为你最得力的科研牛马，安安分分地走正规渠道帮你处理成千上万条数据了。

## 10. 常见疑难杂症与排雷指南

使用过程中难免遇到水土不服，我们总结了最常被问到的“翻车”场景及其急救方案。

**Q1：自动切分出来的片段数量，怎么跟我的字表对不上？**
**症状：** 软件提示某些词“缺失”，或者后面的词全错位了。
**急救：** 最大的可能是长录音里的词连读了没停顿，或者是中间发音人咳嗽了一声被算成了一个词。打开图形界面中间的“可视化切分器”，找到那个多出来的噪音段点右键删掉，或者在没识别出来的波形上点右键强行加一段，排好序后一切就会迎刃而解。

**Q2：提取出来的 F0 曲线全是断点，数据里一堆 0 怎么办？**
**症状：** 图表里坑坑洼洼，均值没法看。
**急救：** 首先看声音底噪是不是太大，或者这段其实是个清辅音（根本没音高）。如果是正常的浊音但依然测不出来，可以谨慎降低 **Pitch Floor** 和 **Voicing Threshold**（清浊判定阈值），再结合声谱图人工复核。如果片段存在嘎裂声、严重不规则振动或弱周期信号，应保留缺失值，不要为了得到连续曲线而强行补齐。

**Q3：音高曲线原本平滑，突然有一段冲上了天（或者掉进了地）？**
**症状：** 典型的倍频/半频错误。
**急救：** 算法把音高算成了一倍或一半。男声容易错高，女声容易错低。如果只错了一两个点，别去调参数了，直接点开“橡皮擦”工具，把那个飞上天的点一擦抹平，是最快的处理办法。如果是大面积错位，尝试降低或升高 Pitch Ceiling。

**Q4：批量匹配时，总是对应错文件？**
**急救：** 检查你的文件名。模糊匹配是很聪明的，但如果你的文件名全是类似 `001-a.wav`, `002-b.wav`，而字表里写的又是完全不同的东西，它肯定会瞎匹配。如果你能保证文件在文件夹里的排序跟字表完全一致，直接用 `order`（顺序匹配）是最高效的防错手段。

**Q5：导出的 TextGrid 怎么长得跟我预想的不一样？**
**急救：** 请注意你当前的模式。批量模式下，软件不会导出一个大 TextGrid，而是会导出一个文件夹，里面每个音频对应一个小 TextGrid。另外，如果是多发音人模式，软件是不支持“整合导出全部发音人的 TextGrid”的，必须分别导出。

**Q6：双击 `.teproj` 启动后，为什么没有弹出“恢复上次异常退出”的提示？**
**急救：** 当你双击 `.teproj` 唤醒软件时，PhonTracer 会优先尝试载入当前工程文件。载入成功后，新工程会替换原工作区；如果外部工程损坏或资源不完整，原自动保存工作区不会被提前删除，仍可用于恢复上次异常退出前的数据。

**Q7：自动保存功能既然开着，为什么还要手动导出 `.teproj`？**
**急救：** 自动保存是“后台保命机制”，备份数据写在用户隐藏目录下，并非用于归档 and 数据分发的格式。要跨设备工作、数据备份或向他人交付成果时，请务必通过“导出项目 (.teproj)”生成无损打包文件。

**Q8：在估计窗口应用了推荐的 F0 范围，为什么我的音节边界没有被自动重新切分？**
**急救：** 这是新版的音节边界保护机制。应用 F0 参数或估计范围时，系统只会更新音高曲线，强制保护您已经辛苦拉好的红蓝边界线不被覆盖。如果您需要根据新参数强制重新切分，请在左侧面板点击“重新自动识别”或在 CLI 输入 `recalculate`，但这会覆盖你的手工修改边界。

**Q9：为什么我的共振峰散点在声谱图上很凌乱，F2 总是跌进 F1 的怀抱（发生错位 swap）？**
**急救：** 共振峰高度敏感于发音人和录音质量。请尝试点击参数面板上的“估计”按钮，优先应用“抗错位档”。此外，对于成年男性，建议将最大共振峰频率限制在 5000 Hz 左右，分析窗长设为 0.025-0.030 秒，并适当提高清浊判定阈值；对于女性和儿童则设为 5500 Hz 甚至更高。如果仅有个别跑飞的点，使用“剔除点”工具单点清除即可。

**Q10：共振峰模式下为什么声谱图右侧没有音高标度，图上也看不到 F0 蓝色音高线？**
**急救：** 共振峰模式致力于母音（元音）空间特征分析，因此右侧的 F0 纵轴及蓝色音高曲线会被自动隐藏。若需要检查基频，请将“分析模式”一键切回“声调/F0”模式。

**Q11：图表编辑器里预览大项目图表或二维 KDE 密度图时非常卡顿？**
**急救：** 二维高斯核密度估计（2D KDE）是极度消耗 CPU 的算法。大样本项目建议关闭左侧面板底部的“实时刷新预览”，集中调整完配色、比例、分组与筛选后，手动点击一次“刷新预览”出图。

**Q12：为什么当科学图表编辑器窗口打开时，系统禁止我导入新的 `.teproj`？**
**急救：** 科学图表编辑器内部对当前项目数据和发音人树进行了高响应性的运行时缓存。如果允许在编辑器打开时导入新工程，很容易引起指针悬空导致崩溃。请关闭图表编辑器窗口，再进行导入操作。

**Q13：在 Windows 系统下，为什么 AudioToolkit 窗口无法直接拖入文件？**
**急救：** 为彻底规避 Windows + Tkinter 多进程架构下偶发的 GIL 锁多线程崩溃，主程序关闭了外部文件直接拖拽的 windnd 入口。请统一使用“添加音频”或“选择长音频”按钮，通过系统的文件资源管理器安全导入文件。列表内部的拖放排序仍可完美使用。
