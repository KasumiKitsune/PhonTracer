# PhonRec 详细操作说明与使用手册事实底稿

> 文档性质：供后续编写正式用户手册时使用的独立事实底稿。
>
> 核对基线：2026-06-22，仓库 `main` 分支，PhonRec 应用版本 `1.3.0`。
>
> 核对范围：React 前端、Tauri/Rust 原生层、PhonTracer Python 分析引擎、共享字表与工程适配代码、相关测试和打包配置。
>
> 重要说明：版本号、下载资产名和界面可能继续变化。正式发布手册前，应重新核对当前 Release、`package.json`、`tauri.conf.json` 与实际安装包。

## 1. PhonRec 是什么

PhonRec 是 PhonTracer 配套的多发音人录音工具，主要用于：

- 导入普通或高级字表；
- 建立多个发音人；
- 按词项逐条录音；
- 播放、丢弃和重录已有录音；
- 对录音做实时或录后质量提示；
- 将录音和字表保存为可与 PhonTracer 主程序互通的 `.teproj` 工程；
- 在独立模式下按发音人、分组和词项批量导出 WAV。

PhonRec 不是 PhonTracer 主程序的内嵌页面，而是单独安装、单独启动的轻量桌面程序。它不在自己的安装包中携带 Python、Praat/Parselmouth、SciPy、Matplotlib 等完整分析依赖。需要语谱图和完整质量分析时，PhonRec 会调用本机已安装的 PhonTracer 分析引擎。

所有录音、字表、工程状态和设置默认保存在本机。程序没有把录音或工程上传到网络的业务代码；启动依赖页上的“打开 PhonTracer 下载页”会打开 GitHub Releases 页面。

## 2. 平台与安装包

当前正式支持范围如下：

| 平台 | PhonRec 安装形式 | 架构边界 | 其他要求 |
| --- | --- | --- | --- |
| Windows | NSIS `.exe` 安装包 | x64、ARM64 | 按当前用户安装 |
| macOS | `.dmg` | 仅 Apple Silicon / ARM64 | macOS 12 或更高版本 |

当前 Release 中，PhonRec 与 PhonTracer 主套件是分开的资产。典型资产名为：

- `PhonRec_1.3.0_x64-setup.exe`
- `PhonRec_1.3.0_arm64-setup.exe`
- `PhonRec_1.3.0_aarch64.dmg`

实际下载时应以目标 Release 页面列出的文件名为准，不要仅根据本节示例猜测版本号。

### 2.1 Windows 安装建议

1. 先确认电脑是常见的 Intel/AMD x64，还是 Windows ARM64。
2. 下载对应架构的 PhonRec 安装包并完成安装。
3. 如果希望使用完整模式，还应安装兼容版本的 PhonTracer 主套件安装版。
4. 启动 PhonRec，等待其自动检测分析引擎。

Windows 完整模式通过 PhonTracer 安装器写入的注册表信息发现 `PhonTracerAnalysisEngine.exe`。仅解压 PhonTracer 便携 ZIP、但没有安装和登记主程序时，PhonRec 正式版可能无法自动发现分析引擎；此时仍可进入独立模式。

### 2.2 macOS 安装建议

1. 下载 Apple Silicon 版本 DMG。
2. 挂载 DMG，将 `PhonRec.app` 拖入“应用程序”。
3. 如果希望使用完整模式，也将兼容的 `PhonTracer.app` 安装到 `/Applications` 或当前用户的 `~/Applications`。
4. 首次录音时，按系统提示允许麦克风权限。

macOS 当前不承诺 Intel Mac，也不提供 Windows 式系统声音回环录制。

## 3. 完整模式与独立模式

PhonRec 每次启动时先尝试连接本机 PhonTracer 分析引擎。根据检测结果，本次运行进入完整模式或由用户选择进入独立模式。

### 3.1 能力对照

| 能力 | 完整模式 | 独立模式 |
| --- | --- | --- |
| 麦克风录音 | 支持 | 支持 |
| Windows 系统声音回环 | 支持 | 支持 |
| 普通 TXT 字表 | 支持 | 支持 |
| CSV 字表 | 支持 | 支持 |
| 高级 PTWL 字表 | 支持 | 支持 |
| `.teproj` 导入与导出 | 支持 | 支持 |
| 本地自动保存 | 支持 | 支持 |
| 音量和削波检查 | 支持 | 支持 |
| 有效语音、噪声、嘎裂声、直流偏移检查 | 支持 | 不提供完整检查 |
| 语谱图 | 支持 | 不支持 |
| F0/F1/F2 叠加分析 | 由完整引擎生成 | 不支持 |
| 文件夹工程导入与导出 | 支持 | 不支持 |
| 按层级批量导出 WAV | 不提供此按钮 | 支持 |

独立模式不是临时演示模式。它会使用与完整模式相同的受控工作区，录音和工程可以正常保存。以后安装或修复 PhonTracer 后，重新启动 PhonRec，完整模式可以继续读取已有工作区。

### 3.2 启动依赖页的状态

启动页可能显示：

- “正在连接 PhonTracer”：正在发现和启动分析引擎；
- “需要安装 PhonTracer”：未发现主程序或关键引擎文件；
- “需要更新 PhonTracer”：引擎协议版本不匹配；
- “分析引擎启动失败”：发现引擎但启动、握手或健康检查失败。

可用操作：

1. **重新检测**：再次发现并启动引擎；
2. **打开 PhonTracer 下载页**：打开最新 Release 页面；
3. **进入独立软件模式**：不等待完整引擎，直接进入本地轻量录音界面；
4. **退出程序**：关闭 PhonRec。

“进入独立软件模式”只影响本次运行，不会把独立模式永久写入设置。要重新尝试完整模式，应退出并重新启动 PhonRec，或在启动依赖页点击“重新检测”。进入主界面后没有即时切换回完整模式的按钮。

完整模式在本机 `127.0.0.1` 随机端口启动分析引擎。除健康检查外的接口均使用本次运行的随机 Bearer token；这不是面向局域网或互联网开放的服务。

## 4. 主界面总览

桌面宽屏界面主要分为四个功能区域。

### 4.1 左侧“发音人与字表”

用于：

- 导入 `.ptwl`、`.txt`、`.csv` 字表；
- 独立模式下粘贴普通字表；
- 查看当前字表名称和词项总数；
- 添加、选择和删除发音人；
- 查看当前发音人的已录制数量和完成百分比；
- 完整模式下快速设置列表角标字段与中心提示字段。

### 4.2 中央录音区

用于：

- 显示当前词项和备注/拼音等辅助字段；
- 调整中央词项字号；
- 播放、暂停和拖动已录音的播放进度；
- 上一条、下一条；
- 开始和停止录音；
- 丢弃当前录音；
- 查看“准备就绪”“录音中”“处理中”状态；
- 切换随机排序；
- 导入工程、导出 WAV、保存工程。

### 4.3 中间右侧“字表词条”

用于：

- 选择全部分组或某一个分组；
- 直接点击任意词项；
- 查看当前、已录制、未录制状态；
- 查看“截断”“偏小”“偏大”“嘎裂”“优质”等简要质量角标。

### 4.4 右侧“检测与配置”

用于：

- 开关实时质量检测；
- 查看待机或录音时的实时音量；
- 查看质量等级、分数和各子项结果；
- 在完整模式中切换波形图与语谱图；
- 选择录音模式；
- 打开设置中心；
- 清空工作区。

窗口宽度较小时，左侧栏会变成可展开抽屉，左上角菜单按钮负责展开或折叠。小屏布局会优先显示全部词项，不应依赖桌面端的分组筛选布局完成导航。

## 5. 第一次使用的推荐流程

建议按以下顺序开始一个新录音项目：

1. 启动 PhonRec，确认进入完整模式还是独立模式。
2. 打开“设置 → 设备与系统”，选择正确的输入设备。
3. 允许麦克风权限，观察实时音量条是否响应。
4. 打开“设置 → 录音与导航”，选择录音模式和快捷键预设。
5. 打开“设置 → 质量检测”，确认需要启用的检查项和严格度。
6. 导入字表。
7. 添加并选择第一个发音人。
8. 录一条测试音，播放检查设备、音量和环境噪声。
9. 正式逐条录音。
10. 中途可随时点击“保存工程”导出 `.teproj` 归档。
11. 完成后再次导出 `.teproj`；独立模式还可额外导出 WAV 文件夹。

本地自动保存用于恢复当前工作区，不等于已经生成可迁移的工程文件。跨设备、交付、备份或长期归档时，必须显式保存 `.teproj`。

## 6. 麦克风权限与输入设备

### 6.1 首次授权

首次请求物理麦克风时，界面会显示授权引导，同时操作系统或 WebView 会弹出权限请求。应在系统提示中选择“允许”。

如果权限被拒绝，界面提供：

- **重新请求权限**；
- **打开系统设置**；
- **稍后处理**。

Windows 在权限被永久拒绝时，程序可能提示重启应用并重置权限记录。确认后，PhonRec 会退出、清理本应用 WebView 的权限数据并重新启动。macOS 可直接打开“隐私与安全性 → 麦克风”进行授权；修改后建议重新启动 PhonRec。

### 6.2 选择输入源

路径：**设置 → 设备与系统 → 麦克风输入源**。

可选来源通常包括：

- 系统默认麦克风；
- WebView 枚举出的各个物理麦克风；
- Windows 下的“系统声音回环（输出设备名称）”。

如果之前保存的设备已断开，启动时会自动回退到“系统默认麦克风”，并更新设置。设备列表没有刷新时，可点击“刷新硬件设备”。

### 6.3 Windows 系统声音回环

回环录音用于录制扬声器或耳机输出，而不是物理麦克风。选择名称以“系统声音回环”开头的设备后：

- 不需要物理麦克风权限；
- 实时音量和波形由 Tauri/Rust 原生层提供；
- 输出设备的多声道信号会混合为单声道；
- 停止录音时再重采样到设置的目标采样率。

系统声音回环当前仅支持 Windows。macOS 设备列表中不会提供该能力。

### 6.4 待机监听

“待机音频输入监听”开启时，程序会在未录音期间维持输入流，以显示实时音量并减少开始录音前的设备准备时间。关闭后，只在点击录制时请求和打开麦克风。

当前实现中，待机监听还要求实时质量检测处于启用状态。关闭总质量检测后，即使“待机音频输入监听”开关仍为开启，也不会维持后台麦克风流。

## 7. 字表准备与导入

### 7.1 支持格式

| 格式 | 用途 | 是否保留高级字段 |
| --- | --- | --- |
| `.txt` | 简单分组和词项 | 仅普通词项结构 |
| `.csv` | 表格交换 | 支持组备注、标签、别名、复核状态和自定义列 |
| `.ptwl` | PhonTracer 高级字表 | 完整保留组级与词项级元数据 |

完整模式和独立模式都支持从文件导入这三种格式。

### 7.2 普通 TXT 格式

普通 TXT 支持 UTF-8 BOM，空行会被忽略。分组标题可以使用：

- `【分组名】`
- `[分组名]`
- `［分组名］`
- `#分组名`

词项可以一行一个，也可以使用空格、Tab、英文逗号、中文逗号或顿号分隔。

示例：

```text
【单字】
妈 麻 马 骂

#双音节
北京 上海
北/京 bro/ther
```

斜杠不会把一行拆成两个独立词项，而是保留在词项标签中，可作为后续多音节边界提示。

没有分组标题的词项会进入“未分组”。

### 7.3 独立模式的“粘贴普通字表”

独立模式左栏额外提供“粘贴普通字表”：

1. 点击“粘贴普通字表”；
2. 直接输入或粘贴普通字表；
3. 界面实时显示识别到的分组数和词项数；
4. 可点击“选择 TXT 文件”载入文本；此按钮只接受 `.txt`；
5. 可点击“复制 AI 整理提示词”，把内置的简单格式整理提示词复制到剪贴板；
6. 点击“确认导入”。

如果没有识别到词项，界面不会导入。此粘贴窗口只处理普通字表；CSV 和 PTWL 应使用主界面的“导入字表”。

### 7.4 CSV 推荐表头

为保证完整模式和独立模式之间兼容，建议使用以下中文表头：

```text
组名,组备注,组标签,词项,词项备注,标签,别名,复核状态,拼音,声调,韵母,实验条件
```

规则：

- 每一行代表一个词项；
- `词项` 是必填列；
- 没有组名时进入默认组或未分组；
- 标签、别名可用分号或逗号分隔；
- 未被识别为标准字段的非空列会保存到词项 `meta`，例如“实验条件”“结构”“词频等级”；
- 带逗号的字段应使用标准 CSV 双引号；
- CSV 中存在未闭合引号时，独立模式会拒绝导入。

英文兼容字段包括 `group`、`group_name`、`label`、`word`、`item`、`group_note`、`group_tags`、`item_note`、`tags`、`aliases`、`metadata_source` 等。正式手册应优先展示中文规范表头，避免用户混用多套别名。

### 7.5 PTWL 高级字表

PTWL 是 JSON 文件，推荐 schema 为 `phontracer.wordlist.v2`。核心结构如下：

```json
{
  "schema": "phontracer.wordlist.v2",
  "title": "普通话声调字表",
  "groups": [
    {
      "id": "group_1",
      "name": "阴平",
      "note": "阴平目标词",
      "tags": ["主测试"],
      "meta": {"实验条件": "A"},
      "items": [
        {
          "id": "item_1",
          "label": "妈",
          "note": "注意自然发音",
          "tags": ["目标词", "单字"],
          "aliases": ["ma"],
          "meta": {"拼音": "ma1", "声调": "1"},
          "metadata_source": "人工填写"
        }
      ]
    }
  ]
}
```

导入时会补齐缺失 ID，并处理重复 ID。组和词项中的 `note`、`tags`、`aliases`、`meta`、`metadata_source` 会随 `.teproj` 往返保存。

### 7.6 导入字表的破坏性影响

导入新字表不是追加操作，而是替换当前字表。确认后会：

- 替换全部分组和词项；
- 清除所有发音人的现有录音记录；
- 清理失去工程引用的受控录音文件；
- 将当前词项重置到第一条。

因此，在已有录音时更换字表前，应先导出 `.teproj` 备份。录音或保存处于“处理中”时，程序禁止更换字表。

### 7.7 拖放导入

主界面支持拖入一个 `.ptwl`、`.txt`、`.csv` 或 `.teproj` 文件。拖入多个文件时，当前实现只处理第一个文件。其他格式会显示“不支持的文件格式”。

## 8. 发音人管理

### 8.1 添加发音人

1. 在左侧“发音人列表”点击“+ 添加”；
2. 输入姓名；
3. 按 Enter，或点击输入框外部完成添加；
4. 新发音人会自动成为当前发音人。

输入为空时，离开输入框会取消添加。每个发音人拥有独立的逐词录音记录，但共用同一份字表。

### 8.2 切换发音人

直接点击发音人名称。程序会保存当前活动发音人状态，并加载目标发音人的录音进度。切换发音人不会删除其他人的录音。

录音开始、停止或保存处理中，发音人切换被禁止，以避免录音写入错误目标。

### 8.3 删除发音人

点击发音人右侧垃圾桶图标，并在确认框中确定。删除会移除该发音人及其全部录音记录；工程状态保存后，对应的无引用托管录音也会被清理。

这是破坏性操作。需要保留时应先导出 `.teproj`。

### 8.4 进度统计

左侧进度卡按当前发音人计算：

- 已录制数：工程中具有有效录音路径的词项数；
- 总数：当前字表全部分组的词项总数；
- 百分比：已录制数除以总数。

切换分组只改变当前显示列表，不改变总进度的分母。

## 9. 词项导航与显示

### 9.1 分组筛选

“当前发音组”可选择：

- `【全部发音组】`；
- 某个具体分组。

选择具体分组后，上下条导航只在该分组的显示列表中移动。切换筛选时，程序会尽量保持当前词项；当前词项不在新列表中时回到第一条。

### 9.2 直接选择词项

点击右侧词项列表中的任意条目即可跳转。普通导航按钮在录音或保存处理中禁用。

如果在正在录音时直接点击另一个词项，当前实现会取消尚未提交的录音，切换到新词项，并尝试在短暂延迟后重新开始录音。被取消的那一段不会保存。为避免误操作，正式录音时仍建议先停止，再切换词项。

### 9.3 随机排序

打开“随机排序录制”后，当前显示词项会被随机打乱。随机排序只改变录音界面的展示顺序：

- 不改变字表源文件中的正式分组和词项顺序；
- 不改变 `.teproj` 中的规范字表顺序；
- 切换分组、字表或布局后可能重新生成显示顺序；
- 程序会尽量按词项 ID 保持当前选中项。

### 9.4 中央辅助字段

中央大字下方可以显示：

- 不显示；
- 词项 `note`；
- 高级字表 `meta` 中的某个字段，例如拼音、声调、韵母或实验条件。

列表角标也可独立选择相同类型的字段。完整模式左栏有快速设置卡；设置中心的“外观与显示”页也提供这些选项。

中央字号可在主界面滑块或设置中心调整。主界面范围为 60–200 px，设置中心范围为 40–200 px。备注/拼音字号随中央字号自动联动，并保持至少 18 px。

## 10. 三种录音模式

### 10.1 点击开关（手动）

1. 点击圆形录音按钮开始；
2. 确认状态变成“录音中”后发音；
3. 再次点击按钮停止；
4. 程序进入“处理中”；
5. 通过保存条件后写入录音，并自动前进到下一条。

标准快捷键预设下可使用空格键开始和停止。

### 10.2 按住录制（对讲机）

1. 鼠标按住或触摸按住录音按钮；
2. 保持按住并发音；
3. 松开、触摸结束或将鼠标移出按钮时停止；
4. 通过保存条件后自动前进。

设备或权限尚未准备完成时，如果用户已经松开，程序会取消本次启动，不会把准备期间误认为有效录音。

按住模式的开始/停止依赖鼠标或触摸按钮。当前全局键盘录音快捷键只处理点击模式和 VAD 模式，不应在按住模式中依赖空格键完成按住语义。

### 10.3 智能 VAD 自动跳转

1. 点击录音按钮或使用录音快捷键启动；
2. 状态显示“静音中（请发音）”；
3. 检测到稳定语音后显示“说话中（静音自动跳转）”；
4. 尾随静音达到阈值后自动停止；
5. 质量条件通过后自动跳到下一条；
6. 短暂停顿后自动开始监听下一条。

VAD 内置保护：

- 未检测到语音约 8 秒后，本次录音丢弃；
- 单次自动录音达到约 15 秒时结束；
- 连续三次未检测到有效语音时，自动流程暂停并提示检查设备或改用手动录音；
- 连续三次质量判定为“需重录”时，自动流程暂停；
- 前两次失败会自动重试当前词项；
- 自动前进和重启之间存在短暂节拍，应以界面真正显示“录音中”为准再发音。

### 10.4 VAD 灵敏度

| 预设 | 语音确认时间 | 尾随静音 | 适用环境 |
| --- | ---: | ---: | --- |
| 稳健型 | 350 ms | 1000 ms | 有环境噪声，宁可稍慢结束 |
| 标准型 | 220 ms | 700 ms | 一般室内录音 |
| 灵敏型 | 150 ms | 450 ms | 非常安静，希望快速断句 |

VAD 是录音启停辅助，不是语音内容正确性认证。噪声、长停顿、极弱发音或持续背景声都可能影响触发。

## 11. 录音提交、重录与数据安全

### 11.1 什么时候真正保存

按下开始时，PhonRec 会立即捕获并锁定：

- 当前发音人；
- 当前词项；
- 当前输入设备；
- 词项的备注、标签、别名和自定义字段。

权限和设备真正就绪后，界面才显示“录音中”。停止麦克风录音时，程序会等待最后一个 WebAudio 缓冲块到达，减少停止瞬间尾部音频丢失。

录音只有在以下条件满足后才提交：

1. 目标发音人和词项仍然存在；
2. WAV 能被正常解析；
3. 启用的质量规则没有给出“需重录”；
4. 音频文件与 `project.json` 状态一起成功落盘。

只有提交成功，当前词项才会标记为已录制并自动前进。

### 11.2 “建议复核”和“需重录”的区别

- **良好 / accept**：保存并前进；
- **建议复核 / review**：仍然保存并前进，但界面显示风险提示；
- **需重录 / retry**：本次音频不提交，不覆盖旧录音，也不把词项标记为完成。

如果正在重录一个已有录音，而新录音被判定为“需重录”或保存失败，原来已接受的录音会保留。

### 11.3 手动重录

选择已经录制的词项，直接再次开始录音。新录音通过保存条件后会替换旧录音；若新录音未通过，旧录音不变。

### 11.4 丢弃录音

1. 选择已录制词项；
2. 点击垃圾桶形“丢弃录音”按钮；
3. 确认。

丢弃会清空当前词项的录音路径、质量结果、录制时间、时长、采样率、声道、格式和来源信息。工程保存后，对应无引用音频会被清理。

## 12. 播放与波形

### 12.1 播放和暂停

已录制词项会启用播放按钮：

- 首次点击加载并播放；
- 再次点击暂停；
- 点击同一条的播放按钮继续；
- 切换发音人、分组或词项时自动停止当前播放并重置播放状态。

标准快捷键预设使用 `R` 播放；其他预设见第 16 节。

### 12.2 播放进度

已录制词项会显示时间轴和“当前时间 / 总时长”。音频加载后可拖动滑块跳转。正在录音、处理，或当前播放器尚未载入这条录音时，时间轴禁用。

### 12.3 波形显示

录音期间显示实时波形；录音完成或切换到已有录音后显示静态波形。静态显示会根据能量估计有效语音区，并在两端各保留约 150 ms 余量，以便让短词波形更清楚。

该裁切只影响界面绘图范围，不会裁掉实际保存的 WAV。

## 13. 语谱图与完整分析

语谱图仅在完整模式可用。选择“语谱图”后：

- 新录音完成时，分析引擎生成 PNG 数据并显示；
- 切换到已有录音时，程序可重新请求分析；
- 图中会尝试叠加 F0、F1、F2 曲线；
- 点击语谱图可放大，点击遮罩或关闭按钮退出。

如果 F0 或共振峰计算失败，后端会尽量保留基础语谱图，而不是让整次录音保存失败。

语谱图和自动声学结果是人工复核辅助，不应在手册中描述成自动质量认证或绝对准确的分析。

## 14. 质量检测

### 14.1 默认规则

首次使用的默认配置为：

| 检查项 | 默认状态 | 完整模式 | 独立模式 |
| --- | --- | --- | --- |
| 有效语音 | 关闭 | 可用 | 不可用 |
| 有效音量 | 开启 | 可用 | 可用 |
| 音频截断 | 开启 | 可用 | 可用 |
| 背景噪声 | 关闭 | 可用 | 不可用 |
| 嘎裂声 | 开启 | 可用 | 不可用 |
| 直流偏移 | 关闭 | 可用 | 不可用 |

每一项可设为“宽松”“标准”“严格”。越严格，越容易要求重录或提示人工复核。

### 14.2 总开关与显示开关

主界面的“实时质量检测”总开关会启用或禁用当前模式可用的规则。独立模式只会控制音量和削波，不会把完整模式专属规则错误地变成可用。

“显示录音质量检测结果”只控制录音后是否显示等级与分数卡，不等于关闭检测。要真正不让质量规则影响保存，应关闭总质量检测或逐项禁用规则。

### 14.3 完整模式判定逻辑

完整模式会计算：

- 录音时长；
- 有效语音毫秒数与语音占比；
- 有效语音 RMS 音量；
- 峰值；
- 背景噪声底和信噪比；
- 削波比例；
- 直流偏移；
- 低频不规则发声比例，用于嘎裂声提示。

典型处理原则：

- 录音过短、有效语音不足、严重削波、音量过小或过大、严重噪声可触发“需重录”；
- 轻微削波、背景噪声偏高、直流偏移、可能的嘎裂声通常触发“建议复核”；
- 嘎裂声只作为复核提示，不会单独自动判定录音失败。

### 14.4 独立模式轻量检测

独立模式只解析标准 16-bit PCM WAV，并计算：

- 有效语音片段的平均音量；
- 峰值和削波比例。

音量超界或严重削波会给出“需重录”；轻微削波给出“建议复核”。没有语谱图、信噪比、有效语音、嘎裂声和直流偏移的完整分析。

### 14.5 结果阅读

结果顶部可能显示：

- `良好 · 100 分`
- `建议复核 · 分数`
- `需重录 · 55 分` 等

完整模式还会显示语音占比和信噪比。下方各项使用文字和指示灯显示“正常”“音量过小”“音量过大”“音频截断”“可能有嘎裂声”“语音不足”“噪声过高”“偏移过高”等。

分数是界面提示值，不是实验数据合格证。最终是否纳入研究仍需人工听辨和复核。

## 15. 工程自动保存、导入与导出

### 15.1 自动保存

以下操作会把最新状态写入本机受控工作区：

- 添加、切换或删除发音人；
- 导入或修改字表；
- 录音成功提交；
- 丢弃录音；
- 导入工程；
- 其他工程状态变更。

自动保存不能关闭。工程状态使用临时文件和原子替换；录音提交失败时会回滚旧音频或保留旧状态。

### 15.2 本机数据位置

Windows：

```text
%LOCALAPPDATA%\KasumiKitsune\PhonRec\
├─ settings.json
└─ workspace\
   ├─ project.json
   └─ audio\...
```

macOS：

```text
~/Library/Application Support/com.kasumikitsune.phonrec/
├─ settings.json
└─ workspace/
   ├─ project.json
   └─ audio/...
```

设置文件与工程工作区分开。清空工程不会恢复默认设置；恢复默认设置也不会删除工程录音。

### 15.3 导入 `.teproj`

1. 点击中央底部“导入”；
2. 选择“导入 `.teproj` 工程文件”；
3. 选择文件；
4. 等待处理；
5. 查看“工程导入详情与警告”。

导入会替换当前工作区，不是与当前工程追加合并。开始导入前，播放停止、输入流关闭。成功后会恢复发音人、活动发音人、字表、录音和元数据。

导入结果窗口可能列出：

- 合并发音人数量；
- 长音频切片数量；
- 缺失字表条目数；
- 降级为未录制的条数；
- 具体兼容性警告。

完整模式通过共享工程适配器处理旧工程、缺失顶层字表、老录音资源与高级字段。独立模式也能从发音人条目重建缺失字表，但其兼容报告通常只返回零统计和空警告。

两种模式在提交导入前都会先暂存和验证工程。导入失败时应保留原工作区，而不是留下半导入状态。

### 15.4 `.teproj` 安全限制

独立模式会拒绝：

- 无效 ZIP；
- 缺少 `project.json`；
- 不支持的工程版本；
- 路径穿越、绝对路径；
- 符号链接；
- Windows 保留文件名和跨平台非法字符；
- 大小写折叠后重复的成员；
- 超过成员数或解压大小上限的归档；
- `project.json` 引用不存在或越界的音频资源。

当前独立导入上限为 10,000 个归档成员、单成员 2 GiB、解压总量 20 GiB。这些上限是安全边界，不是推荐工程规模。

### 15.5 保存 `.teproj`

1. 点击“保存工程”；
2. 程序先把当前状态同步到工作区；
3. 选择目标路径；
4. 输入或确认文件名；
5. 等待“工程已成功保存”。

没有任何发音人时禁止导出。默认文件名会清理 Windows 非法字符，并自动补 `.teproj` 扩展名。

`.teproj` 是正式的可迁移归档，包含 `project.json` 和工程引用的音频资源。应把它作为备份、迁移和交付的首选格式。

### 15.6 完整模式的文件夹工程

完整模式可在“设置 → 工程与保存”把默认格式改为“外部工程文件夹目录（解包暴露模式）”。保存后典型结构为：

```text
目标目录/
├─ .phonrec-project.json
├─ project.json
├─ audio/
│  └─ 发音人目录/
│     └─ 编号_词项__词项ID.wav
├─ wordlist/
│  └─ wordlist.ptwl
└─ logs/
   ├─ recordings.csv
   └─ export.json
```

`recordings.csv` 包含发音人、分组、词项、相对音频路径、录制时间、时长、采样率、来源设备和质量摘要。

目标目录必须满足下列条件之一：

- 尚不存在；
- 是空目录；
- 是带有效 `.phonrec-project.json` 标记的既有 PhonRec 文件夹工程。

程序不会把工程直接覆盖到一个任意的非空普通目录。导入文件夹工程也只接受带合法标记、具有 `project.json` 且资源路径安全的目录。

文件夹工程便于人工检查文件，但正式归档和跨设备交换仍优先使用 `.teproj`。

## 16. 独立模式批量导出 WAV

独立模式中央底部显示“导出 WAV”。完整模式不显示该按钮，应使用工程导出。

操作：

1. 可先在“设置 → 保存与导出”设置默认 WAV 导出目录；
2. 点击“导出 WAV”；
3. 未设置默认目录时选择一个父目录；
4. 等待导出完成提示；
5. 查看成功条数、跳过条数和最终目录。

每次导出都会新建目录，不覆盖以前的结果：

```text
PhonRec_WAV_YYYYMMDD_HHMMSS/
├─ 01_发音人甲/
│  ├─ 01_阴平/
│  │  ├─ 001_妈.wav
│  │  └─ 002_八.wav
│  └─ 02_阳平/
└─ 02_发音人乙/
```

目录已存在时自动追加 `_2`、`_3` 等后缀。发音人、分组和词项名称中的非法文件名字符会被替换或清理。

没有录音路径、资源缺失或无法安全读取的词项会计入“跳过”，不会生成空 WAV。导出完成后应检查成功与跳过数量是否符合预期。

## 17. 快捷键

快捷键只在焦点不位于输入框、按钮、下拉框、可编辑区域或弹窗时生效。

| 预设 | 录音/停止 | 上一条 | 下一条 | 播放 |
| --- | --- | --- | --- | --- |
| 标准 | Space | ← | → | R |
| 左手 | Space | A | D | S |
| 右手 | Enter / 小键盘 Enter | J | L | K |
| 禁用 | 无 | 无 | 无 | 无 |

注意：

- 录音或处理中，导航和播放快捷键被忽略；
- 点击模式和 VAD 模式支持键盘开始/停止；
- 按住模式应使用鼠标或触摸录音按钮；
- 主界面底部的简短提示可能仍显示标准键位，准确绑定以“设置 → 录音与导航 → 当前按键映射”为准；
- 可关闭“操作提示显示”，但不会关闭快捷键本身。

## 18. 设置中心逐项说明

### 18.1 外观与显示

- **外观主题**：浅色、深色、跟随系统；
- **颜色主题**：深蓝、红、绿、蓝、紫；
- **界面缩放**：90%、100%、110%、125%；
- **布局密度**：紧凑、标准、宽松；
- **界面过渡动效**：开或关；
- **默认图形**：完整模式可选波形或语谱图，独立模式锁定波形；
- **字表角标字段**：无、note 或高级元数据字段；
- **中心提示显示字段**：无、note 或高级元数据字段；
- **显示录音质量检测结果**：只隐藏或显示结果卡；
- **中央字词字号**：40–200 px。

选择“跟随系统”后，系统深浅色变化会即时应用。

### 18.2 录音与导航

- **录音模式**：点击开关、按住录制、智能 VAD；
- **录制顺序**：字表顺序、随机乱序；
- **待机音频输入监听**：持续显示输入音量，或仅录音时打开设备；
- **VAD 灵敏度**：稳健、标准、灵敏；
- **快捷键预设**：标准、左手、右手、禁用；
- **操作提示显示**：控制中央录音区底部的提示条。

### 18.3 质量检测

- 每项规则可独立开关；
- 每项可选择宽松、标准、严格；
- 顶部总开关启用或关闭当前模式可用的全部规则；
- 独立模式中完整分析项会显示“完整模式可用”并禁用。

### 18.4 工程与保存 / 保存与导出

完整模式：

- 默认工程保存格式：`.teproj` 或文件夹；
- 文件夹模式的默认目标路径；
- 默认 `.teproj` 文件名。

独立模式：

- 说明录音自动保存到受控工作区；
- 默认 WAV 导出目录。

完整模式的 `folder_path` 与独立模式的 `wav_export_path` 是两套独立设置，互不覆盖。

### 18.5 设备与系统

- 输入设备；
- 刷新设备；
- 目标采样率；
- 单声道只读提示；
- 16-bit 线性 PCM WAV 只读提示；
- 麦克风权限状态；
- 重新请求权限；
- 打开系统隐私设置。

目标采样率可选：

- 16 kHz：适合语音识别和分析；
- 44.1 kHz：常见高保真音频；
- 48 kHz：当前默认值，常见于系统和广播音频。

录音会读取设备的真实采样率，停止时重采样到用户选择的目标值；保存格式固定为单声道 16-bit PCM WAV。

### 18.6 恢复默认

设置中心左下角“恢复默认”会要求确认。恢复后只重置设置，不清空发音人、字表或录音。

主要默认值：

- 浅色主题；
- 100% 缩放；
- 标准密度；
- 波形图；
- 字表顺序；
- 点击开关录音；
- 系统默认麦克风；
- 48 kHz；
- `.teproj` 保存；
- 标准 VAD；
- 标准快捷键；
- 待机监听开启；
- 中央字号 120 px；
- 质量检测开启，但仅默认启用部分子项。

设置变更会立即写入 `settings.json`。如果保存失败，界面会恢复修改前的设置并提示错误。

## 19. 清空工作区

点击右侧底部“清空工作区”，确认后会：

- 删除当前受控工作区中的工程状态和录音；
- 清空发音人；
- 清空字表；
- 重置当前词项、波形、语谱图和质量显示；
- 保留个人设置，并重新应用已保存设置。

清空工作区不可通过界面撤销。操作前应先导出 `.teproj`。

## 20. 常见故障与处理

### 20.1 启动时找不到 PhonTracer

可能原因：

- 未安装主程序；
- Windows 只解压了便携 ZIP，没有安装注册信息；
- 主程序版本过旧；
- 引擎文件缺失；
- macOS 的 `PhonTracer.app` 不在系统可发现位置。

处理：

1. 点击“重新检测”；
2. 安装或更新匹配版本的 PhonTracer；
3. 重新启动 PhonRec；
4. 暂时只录音时进入独立模式。

### 20.2 麦克风没有声音

1. 打开“设置 → 设备与系统”；
2. 检查是否选错设备；
3. 点击“刷新硬件设备”；
4. 查看麦克风权限状态；
5. 点击“重新请求并检测权限”；
6. 打开系统隐私设置；
7. 观察实时音量条；
8. 关闭其他可能独占设备的音频软件后重试。

### 20.3 回环录音没有声音

- 确认系统是 Windows；
- 确认选择的是正在实际播放声音的输出设备；
- 在开始录音前先播放系统音频并观察实时音量；
- 输出设备切换后刷新设备列表并重新选择。

### 20.4 VAD 一直不开始或不停

- 太安静或麦克风增益过低：检查音量，换“灵敏型”；
- 背景噪声持续：换“稳健型”，改善环境；
- 说话前等待状态真正进入“录音中”；
- 连续三次失败后，先改用点击模式检查设备；
- 长音、持续背景声或没有尾随静音时，VAD 会等到最大时长保护。

### 20.5 录音后没有标记完成

可能原因：

- 质量结果为“需重录”；
- 当前发音人或词项在录音过程中被删除或失效；
- 工作区写入失败；
- WAV 解析失败；
- 麦克风没有产生有效采样。

先查看质量提示和错误弹窗，再播放旧录音确认是否仍保留。质量重试不会覆盖已有的合格录音。

### 20.6 导入字表后旧录音消失

这是当前设计：导入新字表会替换字表并清除所有发音人的录音记录。应从导入前保存的 `.teproj` 恢复。

### 20.7 `.teproj` 导入失败

检查：

- 文件是否为有效 `.teproj` ZIP；
- 是否包含 `project.json`；
- 工程版本是否为支持的 1.0；
- 音频资源是否缺失；
- 是否包含越界、绝对、非法或重复路径；
- 是否来自不完整拷贝。

导入失败后，原工作区应仍然可用。不要先清空工作区再尝试导入唯一备份。

### 20.8 文件夹工程不能导入

文件夹工程只在完整模式可用，并且必须包含合法 `.phonrec-project.json` 标记和 `project.json`。任意手工组织的 WAV 文件夹不能直接作为文件夹工程导入。

### 20.9 WAV 导出有跳过项

跳过通常表示：

- 词项尚未录制；
- 工程记录有路径，但文件不存在；
- 资源路径不安全；
- 音频无法读取。

对比导出提示中的成功数、跳过数和当前发音人进度，定位缺失词项。

## 21. 不可逆操作与备份建议

以下操作具有删除或替换效果：

| 操作 | 影响 | 建议 |
| --- | --- | --- |
| 导入新字表 | 替换字表并清除所有录音记录 | 先导出 `.teproj` |
| 导入 `.teproj` | 替换当前工作区 | 先导出当前工程 |
| 删除发音人 | 删除该发音人的全部录音 | 先备份工程 |
| 丢弃录音 | 删除当前词项录音引用并清理资源 | 先播放确认 |
| 清空工作区 | 删除全部发音人、字表和录音 | 必须先导出 `.teproj` |
| 合格录音上再次录音 | 新录音通过后替换旧录音 | 先试听；失败不会覆盖旧录音 |

推荐备份策略：

1. 每次正式录音开始前导出一个空框架 `.teproj`；
2. 每完成一个发音人导出一次；
3. 当天结束时另存带日期的 `.teproj`；
4. 不要只复制 `project.json`，因为录音资源在 `audio/`；
5. 不要把本机自动保存工作区当作唯一长期备份。

## 25. 质量与 VAD 阈值附录

本节用于帮助后续手册作者解释“宽松、标准、严格”的真实差异。普通用户手册不一定需要展示全部数字，但不得写出与这些逻辑相反的说明。

### 25.1 完整模式有效语音阈值

| 档位 | 最少有效语音 | 最低语音占比 |
| --- | ---: | ---: |
| 宽松 | 100 ms | 4% |
| 标准 | 180 ms | 8% |
| 严格 | 300 ms | 15% |

启用“有效语音”规则时，整段录音短于 300 ms 也会判定为“录音过短”。有效语音判断使用按录音自适应的能量阈值，不是固定音量门限。

### 25.2 完整模式和独立模式的音量阈值

| 档位 | 过小阈值 | 过大阈值 |
| --- | ---: | ---: |
| 宽松 | 小于 -40 dBFS | 大于 -3 dBFS |
| 标准 | 小于 -35 dBFS | 大于 -6 dBFS |
| 严格 | 小于 -30 dBFS | 大于 -9 dBFS |

完整模式还把峰值高于约 -0.15 dBFS 视为音量过大。这里分析的是估计出的有效语音片段，不是把整段首尾静音一起平均。

### 25.3 削波阈值

下表中的数值是达到数字满幅附近的采样占总采样的比例。

| 档位 | 高于此比例提示复核 | 达到此比例要求重录 |
| --- | ---: | ---: |
| 宽松 | 0.3% | 1% |
| 标准 | 0.01% | 0.3% |
| 严格 | 任何检测到的削波 | 0.1% |

“高于复核阈值但未达到重录阈值”属于轻微削波，录音仍保存；达到重录阈值则不提交本次录音。

### 25.4 完整模式信噪比阈值

| 档位 | 低于此值要求重录 | 此区间建议复核 |
| --- | ---: | ---: |
| 宽松 | 3 dB | 3–8 dB |
| 标准 | 6 dB | 6–12 dB |
| 严格 | 10 dB | 10–16 dB |

只有启用“背景噪声”规则时这些阈值才影响判定。

### 25.5 完整模式嘎裂声与直流偏移

| 规则 | 宽松 | 标准 | 严格 | 判定影响 |
| --- | ---: | ---: | ---: | --- |
| 可能的嘎裂声比例 | 25% | 15% | 8% | 建议复核 |
| 直流偏移绝对值 | 0.05 | 0.03 | 0.015 | 建议复核 |

嘎裂声检测使用短时自相关寻找约 45–85 Hz 的低频发声帧，只能作为提示，不能代替语音学人工判断。

### 25.6 质量分数

完整模式从 100 分开始：

- 只要出现一项“需重录”问题，扣 45 分；
- 每项“建议复核”问题扣 8 分，复核项合计最多扣 30 分；
- 最低为 0 分。

独立模式出现需重录问题时分数固定为 55；只有轻微削波等复核项时，从 100 分按每项 8 分扣减。

分数只用于快速浏览，不适合直接作为论文中的连续型质量变量。

### 25.7 VAD 内部阈值

除第 10.4 节列出的确认与尾随静音时间外，VAD 还会根据待机噪声动态估计门限：

- 初始最低噪声底约为 RMS 0.0015；
- 标准预设进入语音的噪声倍率约为 3.2，退出倍率约为 1.8；
- 稳健预设约为 3.5 / 2.0；
- 灵敏预设约为 2.8 / 1.5；
- 没有语音约 8 秒超时；
- VAD 自动录音约 15 秒强制结束。

这些参数会受设备增益、环境噪声和发音方式影响。手册应鼓励用户先做一条测试录音，而不是只按数字选择预设。

### 25.8 独立模式 WAV 解析边界

独立运行时只接受具有标准 RIFF/WAVE 结构的 16-bit PCM WAV。录音可以来自 16 kHz、44.1 kHz 或 48 kHz 设置；多声道输入在采集阶段转为单声道。独立运行时对单条 WAV 的解析保护上限为 10 分钟，日常逐词录音不应接近此上限。
