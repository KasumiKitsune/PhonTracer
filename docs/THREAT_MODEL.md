# 自定义脚本与本地工程威胁模型

最后核对日期：2026-06-21

## 目标与非目标

当前脚本系统的目标是为研究者自己的脚本提供受控 API，降低误写文件、误改工程和意外调用系统功能的风险。它不是针对恶意代码的操作系统级沙箱，也不提供进程、容器、虚拟机或独立用户权限隔离。

因此：**只运行自己编写、已审阅或来自可信来源的脚本。** 对不可信脚本，仅靠当前静态检查不足以保证宿主和数据安全。

## 需要保护的资产

- 原始录音、切分音频、发音人身份与研究元数据；
- `.teproj`、TextGrid、分析缓存和人工修正；
- 用户目录中的其他文件与凭据；
- 主程序可用性、内存、CPU 与磁盘空间；
- 研究结果的可复现性和脚本执行记录完整性。

## 信任边界与输入

| 输入 | 默认信任级别 | 主要风险 |
|---|---|---|
| 自定义 Python 脚本 | 仅可信作者 | 代码执行、资源耗尽、逃逸受限 API |
| `.teproj` | 不可信文件 | ZIP 路径穿越、压缩炸弹、资源替换、损坏缓存 |
| `.ptwl`、CSV、TXT | 不可信数据 | 超大输入、字段混淆、路径或显示注入 |
| 音频、TextGrid | 不可信媒体/文本 | 解析器缺陷、超大资源、异常数值 |
| PhonRec 本机 HTTP | 仅同机进程 | 未授权调用、端口探测、会话令牌泄露 |

## 当前控制措施

### 脚本执行

- 使用 AST 拦截不允许的导入、动态执行、文件打开和直接写出调用；
- 只暴露有限内置函数和导入根模块；
- 不允许相对导入，不允许 `parselmouth.praat`；
- 图表和表格通过 `ctx.figure(...)`、`ctx.table(...)` 返回；
- 工程写回通过 `ProjectPatchResult` 的允许操作，由宿主再次校验并事务化写入；
- 使用超时和取消提示减少无界计算造成的界面阻塞；
- 执行记录可归档进工程，帮助复现。

### 工程导入

- ZIP 成员数量和解压大小设有上限；
- 拒绝绝对路径、`..`、符号链接、大小写折叠重复和跨平台非法路径；
- `project.json` 单独限额并验证顶层类型与格式版本；
- 资源必须位于 `audio/` 或 `data/`，并验证实际存在和缓存关键数组；
- 在临时工作区验证，成功后交换，失败时回滚。

### PhonRec 引擎

- 绑定 `127.0.0.1` 随机端口，不作为远程服务公开；
- Tauri 启动引擎时生成会话令牌；
- 除健康检查外的业务接口要求 Bearer 令牌；
- 前端 CSP 限制可连接的来源和资源类型。

## 已知残余风险

- Python 对象模型和允许导入的科学库表面积很大，静态黑名单无法证明恶意脚本不可逃逸；
- 超时不会强制终止正在运行的 Python 线程，脚本可能在返回超时后继续消耗资源；
- 允许库可能间接访问本地资源或触发高成本计算；
- AST 按函数名拦截输出 API 可能存在漏报，也可能对无害同名方法产生误报；
- 大型但未越过硬上限的工程仍可能占用显著内存、磁盘和解析时间；
- 回环接口仍可能被同一账户下的本地恶意进程观察或竞争，令牌只降低误调用和普通网页调用风险；
- 未签名或未公证的安装包不能抵御下载渠道被替换。

## 维护要求

1. 不要在 UI 或文档中把当前机制表述成“安全运行任意第三方代码”。
2. 新增允许模块、内置函数或工程操作前，必须评估文件、网络、进程、反射和资源耗尽能力。
3. 安全检查与 API 变更必须增加允许路径和拒绝路径测试。
4. 工程格式变更必须保留临时导入、资源约束与失败回滚。
5. 若要支持真正不可信脚本，应迁移到独立进程，并配合操作系统权限、资源配额、文件系统白名单和可强制终止机制。
6. 漏洞报告按 [SECURITY.md](../SECURITY.md) 私下处理。
