# 安全策略

## 支持范围

安全修复优先面向最新正式版本与当前 `main`。旧版本可能只收到升级建议，不保证单独回补。

## 私下报告漏洞

请优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私有通道。若该入口不可用，可先创建一条不含漏洞细节、复现附件或敏感数据的普通 Issue，请维护者提供私下沟通方式。

报告建议包含：

- 受影响版本、组件、系统与架构；
- 风险类型和可能影响；
- 最小复现步骤或概念验证；
- 是否需要打开恶意 `.teproj`、`.ptwl`、脚本或其他文件；
- 可行的缓解与修复建议。

在维护者确认修复或协调披露时间前，请不要公开利用代码、恶意样本、令牌或可识别研究数据。

## 重点安全边界

- `.teproj` 与 `.ptwl` 属于不可信输入，应只从可信来源打开；
- Toolkit 自定义脚本是进程内受控执行，不是操作系统或容器级安全隔离；不要运行不可信脚本；
- PhonRec 分析引擎只应绑定本机回环地址，并为业务接口使用随机会话令牌；
- 发布产物目前不等同于经过代码签名或公证，下载者应从官方 Releases 获取并核对发布说明。

脚本模型的详细限制见 [威胁模型](docs/THREAT_MODEL.md)。
